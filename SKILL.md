# AI Strategy Consulting

## 0 · 总览

本 skill 的核心是**一条 AI 战略咨询从零到交付的工作流**，配套**一组防止 AI 战略报告翻车的纪律清单**。沉淀自多份咨询文档体系 + 多轮 codex 外审 + 100+ 页国企客户战略稿的实战教训。

**自包含承诺**：本 skill **零外部 skill 依赖** —— 所有方法论、模板、检查脚本均内嵌。Python 脚本仅用标准库(sys/re/pathlib),无需 pip install。HTML 图表模板在浏览器直接打开就能看,无需任何前端框架或截图服务。

**关键差异化(vs 其他咨询写作类 skill)**：本 skill 聚焦 **AI 战略咨询** 特定场景,覆盖从信息采集 → 分析筛选 → 报告起草 → 外审清稿的完整工作流(不只写作)。

**关键差异化**：
1. 五维信息采集（含 AI 战略独有的"跨行业 AI 实践"维度）
2. 价值驱动设计四问（避免技术堆砌）
3. 三层价值承接 L1+L2+L3+L4（避免单一 ROI 视角）
4. 13+ AI 战略报告反模式（含跨客户内容入稿、国际公司引文越级、自证式论据、机会成本威胁等独家踩坑）
5. 战略判断分层表达（基座/行业/应用三层 · 避免"路径被堵死"偷懒表达）
6. 精益启动 ROI 模板（适配国企/工程基建/政府等中小预算客户）
7. **审稿四层认知架构**(L1 病因 / L2 前提 / L3 全局 / L4 结构 · 让审稿者懂"为什么"而不只是"做什么",见 [11-review-thinking-model](references/11-review-thinking-model.md))
8. **陌生行业摸底框架**(任意行业 1-2 小时建立够用认知,**不依赖预制 checklist**,见 [12-unfamiliar-industry-framework](references/12-unfamiliar-industry-framework.md))

## 0.5 · 通用前置 · 陌生行业判断(Mode L / Mode D 启动前必做)

任何模式启动前,先判断目标行业是否在 [01 §8 样板库](references/01-information-gathering.md#8-采集-checklist按行业-4-套)(国企央企 / 民营上市 / 创业 / 政府事业 / 医疗 / 教育 / 制造 / 物流 / 零售 / 农业):

- **在样板库** → 直接用现成 checklist,跳过本前置
- **不在样板库**(如宠物 / 殡葬 / 印刷 / 民营养老 / 二手奢侈品 / 任意长尾业态) → 先跑 [12-unfamiliar-industry-framework](references/12-unfamiliar-industry-framework.md) 的 5 步法
  - 产出 A · 行业知识背景(2-3 页)
  - 产出 B · 该行业 AI 应用框架(2-3 页)
  - 用户校验三问(痛点对不对 / 价值链准不准 / 首发场景靠不靠谱)
  - 通过后再进入 Mode L / Mode D

**为什么必须有这个前置**:行业有几千个,样板库永远填不全。框架优先 + 实例补充——任何陌生行业用框架 1-2 小时建立够用认知,优于硬把陌生行业塞进相近样板。

## 1 · 四种工作模式（先选模式）

启动前必须明确当前在哪个模式，模式不同则后续步骤不同。

| 模式 | 触发场景 | AI 生成 | 用户审稿(瓶颈) | 最小输入 | 关键产出 |
|---|---|---|---|---|---|
| **Mode L · Lite 个人/团队探索** | 行业从业者自问"AI 在我们行业能做什么" / 新行业可行性快查 | 0.5-1.5 小时 | 0.5 天 | 行业 + 角色 + 一句痛点 | 5-10 页 Markdown:行业摸底 + 机会池 + Top 3 方向 |
| **Mode D · Discovery 信息采集与诊断** | 新客户接入 / 战略议题刚确立 / 现有素材不足 | 1-2 小时 | 0.5 天 | 客户名 + 行业 + 战略议题 | 五维输入清单 + 客户画像 + 竞争格局 + 政策环境 + 学术前沿 |
| **Mode P · Planning 分析筛选与定向** | 已采集足够信息 / 待出方向 | 0.5-1 小时 | 0.5 天 | Discovery 全部产出 | 机会池 + 五把筛子 + Top N 方向 + 价值承接映射 |
| **Mode R · Reporting 报告起草与修订** | 已确定方向 / 进入交付稿阶段 | 2-3 小时 | 1-2 天 | Planning 产出 + 客户调性 | 主报告 + 一页纸 + PPT + 附件 + 外审清稿 |

**典型走向**:
- 个人/团队探索:**Mode L 一站式**(0.5-1 天落地)
- 企业级正式稿:**Mode D → P → R**(AI 生成 3-6 小时 + 用户审稿 2-3 天 = **整体 2-4 天落地**)

**关键认知 · 瓶颈在审稿不在生成**:
- AI 生成可一气呵成,不要按传统人工咨询周期(数周)预期工时
- 但务必给审稿留足天数 —— 每天 3-4 小时的有效审稿时间,2-3 天才能把 60-100 页主报告过完
- 跳过任一企业级阶段:Mode P 跳过会导致报告"只罗列方向,没有论证";Mode D 跳过会导致"自说自话,缺乏证据基础"

## 2 · Mode L · Lite 个人/团队探索

### 2.1 适用场景

任意一种以下情形,优先用 Mode L 而非 Mode D-P-R 链路:

- 行业从业者自问:"AI 在我们行业能做什么?"
- 团队拿到一个新行业方向,想 1-2 小时内出个**初步可行性判断**
- 没有正式客户咨询任务,只是探索 / 学习 / 内部分享
- 时间窗紧(不到 1 天)且不需要 60-100 页主报告

**反过来:不适用 Mode L 的场景**——已签约的咨询交付稿 / 客户终审稿 / 60-100 页正式战略报告 → 走 Mode D-P-R。

### 2.2 输入(三件就够)

| 项 | 内容 | 例子 |
|---|---|---|
| 行业 | 50 字内说明你想探索什么行业 | "民营养老院"、"宠物医院连锁"、"二手奢侈品鉴定平台" |
| 角色 | 你是谁 / 与该行业的关系 | "我是从业者(养老院主任 5 年)"、"我家人开宠物医院"、"我朋友想做该业态创业" |
| 一句痛点 | 你最直觉的"AI 也许能解决"的痛点 | "新人培训慢,半年才能独立带床位"、"医生流动性高,经验留不住"、"鉴定真伪靠老师傅,人手不够" |

如果连"一句痛点"都说不清 → 先去看 [12-unfamiliar-industry-framework §4 5 步法](references/12-unfamiliar-industry-framework.md#4--5-步法详解) 步骤 4(从业者论坛),找到 5-10 条具体痛点再回来。

### 2.3 工作流(分支两条)

```
输入:行业 + 角色 + 一句痛点
   │
   ▼
[判断该行业是否在 §0.5 样板库]
   │
   ├── 在样板库
   │   └── 直接调用 [01 §8](references/01-information-gathering.md#8-采集-checklist按行业-4-套) 的对应 checklist 走快速版
   │
   └── 不在样板库(更常见)
       └── 跑 [12-unfamiliar-industry-framework §4](references/12-unfamiliar-industry-framework.md#4--5-步法详解)
           5 步法 → 产出 A 行业知识背景 + 产出 B AI 应用框架
           (1-1.5 小时)
   ↓
   用户校验三问(30 分钟):痛点对不对 / 价值链准不准 / 首发场景靠不靠谱
   ↓
   通过
   ↓
[五把筛子](references/02-frameworks.md)
   │  对产出 B §一 的 8-15 个候选场景打分
   │  每筛 1-5 分,过 18 分的入 Top 名单
   ↓
[Top 3 方向 · 价值四问 + 三层承接简版]
   │  每个方向回答:
   │  - 解决什么具体痛点
   │  - 提效多少(数量级即可,带【推断】/【假设】标签)
   │  - 要做的第一步是什么(本周 / 本月可启动的最小动作)
   │  - 边界:这个方向不能解决什么
   ↓
[输出 5-10 页 Markdown · 见 assets/lite-mode-template.md]
```

### 2.4 产出物 · 5-10 页 Markdown

唯一产出文件:`AI_探索_[行业名]_[日期].md`(模板见 [assets/lite-mode-template.md](assets/lite-mode-template.md))

结构骨架:
```
# AI 应用探索 · [行业名]

## 一、行业速览(0.5-1 页)
   - 来自陌生行业框架产出 A 的精简版

## 二、核心痛点(0.5 页)
   - 5 个最痛的痛点,用从业者原话(来自 5 步法步骤 4)

## 三、AI 候选场景 8-15 个(1-2 页)
   - 表格形式:场景 / 对应痛点 / 技术路径 / 跨行业标杆 / 五把筛子打分

## 四、Top 3 推荐(2-3 页)
   - 每个 0.7-1 页,价值四问 + 第一步建议

## 五、不做什么(0.5 页)
   - 应用边界 + 不建议方向 + 理由

## 六、第一步行动建议(0.5-1 页)
   - 本周可做的 3 件事
   - 1 个月内可验证的 1 个 MVP

## 七、参考资料(0.5 页)
   - 5 步法跑出来的来源链接 + 跨行业标杆原文
```

### 2.5 时间预算

| 阶段 | AI 生成 | 用户参与 |
|---|---|---|
| 陌生行业摸底(若需要)| 1-1.5 小时 | 30 分钟校验 |
| 五把筛子 + Top 3 | 30-60 分钟 | — |
| 5-10 页 Markdown 起草 | 30 分钟 | — |
| 用户最终审稿 | — | 30 分钟 |
| **整体** | **2-3 小时** | **1 小时** |

**整体落地周期**:0.5-1 天(对比 Mode D-P-R 的 2-4 天)

### 2.6 Mode L 的纪律(必读)

Mode L 比 Mode D-P-R 轻量,但**核心纪律不打折**:
- ✅ **三标签** —— 即使是探索文档,每个判断也要标【事实】/【推断】/【假设】
- ✅ **价值四问** —— Top 3 必过四问,简版即可
- ✅ **不做什么** —— 必须有应用边界章节(咨询价值在"清晰说不")
- ✅ **反 AI 腔** —— `python scripts/ai_slop_check.py` 跑一遍

不必做的(Mode L 豁免):
- ❌ ROI 详细测算(精益启动模板太重,Mode L 用"提效数量级"即可)
- ❌ 11 章主报告骨架(用 5-10 页 Markdown 模板)
- ❌ 一页纸 + PPT + 6 类附件(Mode L 不出这些)
- ❌ 外审 6 步法(Mode L 用作探索,不做对外交付)

### 2.7 Mode L → Mode D-P-R 的升级

如果探索结果证明"值得做正式战略报告",从 Mode L 升级到 Mode D-P-R:
- Mode L 产出的"行业速览"和"核心痛点"成为 Mode D 维度三/维度一的输入
- Mode L 产出的"AI 候选场景"成为 Mode P 机会池的起点
- Mode L 产出的"Top 3 推荐"成为 Mode P 五把筛子已初评的基础
- Mode L 产出的"不做什么"成为 Mode R §8 风险与边界的草稿

升级路径让 Mode L 的工作不浪费 —— **Mode L 是 Mode D-P-R 的轻量预演,不是替代品**。

## 3 · Mode D · Discovery 信息采集与诊断

### 3.1 五维信息采集（必填全部五维）

| 维度 | 内容 | 资料来源 | 行业适配 |
|---|---|---|---|
| **客户自身** | 公司基本盘、业务结构、技术现状、战略意图、决策链 | 官网、年报、招股书、新闻、招聘 JD、组织架构 | 不同行业披露程度差异大；国企看年报+党建报道，民企看融资+招聘 |
| **政策环境** | 国家/部委/行业政策窗口、考核节点、合规要求 | 国务院政策库、部委公告、地方文件、合规标准 | AI 战略必查：网信办、工信部、发改委、行业部委 |
| **行业现状** | 行业规模、增速、价值链、痛点、数字化成熟度 | 行业研究报告（艾瑞/赛迪/IDC/Gartner）、协会报告、龙头年报 | 工程基建查住建部+交通部，金融查央行+证监会，制造查工信部 |
| **学术前沿** | 行业 AI 应用方向的 SOTA、可行性边界、典型论文 | arXiv、Semantic Scholar、Papers with Code、行业顶会、IEEE/ACM | AI 战略必看：近 18 个月顶会论文 + 工业级落地报告 |
| **跨行业 AI 实践** | 其他行业类似场景的 AI 落地案例（成功 + 失败） | OpenAI/Anthropic/BCG/McKinsey/MIT Sloan 公开报告、Palantir/Snowflake 客户案例 | 不限行业；跨行业类比是 AI 战略的常用论证手段 |

**详细方法见 [references/01-information-gathering.md](references/01-information-gathering.md)**（含每维的检索关键词、信息源 URL 列表、采集 checklist）

### 3.2 信息缺口检测（何时主动问用户）

**强制规则：信息缺口出现时必须问用户，不允许猜测填充。**

下列三类缺口必须问：

| 缺口类型 | 触发情形 | 必问问题示例 |
|---|---|---|
| **客户私域信息** | 公开资料无法回答的客户内部数据（人员规模 / 实际营收 / 业务真实占比 / 内部 IT 现状 / 决策链权力关系） | "客户公开年报营收 X 亿，但业务结构未细分。请提供：A 业务收入 / 工程师人数 / 现有 IT 投入比例" |
| **战略意图模糊** | 客户战略意图、读者画像、决策时间窗、报告用途存在多种可能解读 | "本报告是给董事会汇报、政府备案、还是内部使用？决策时间窗是？是否要求一页纸 + 主报告 + PPT 三件套？" |
| **行业专有约束** | 行业法规、客户细分、合规要求超出公开资料覆盖度 | "客户所在 X 业务子赛道，是否有未公开的牌照/资质要求？是否受到 Y 监管？" |

**问的频率纪律**：
- 一次问 ≤ 3 个问题（避免轰炸用户）
- 优先级排序：能直接卡住后续工作的问最先
- 用户深夜回答的指令（22:00 后）特别确认，避免歧义执行

### 3.3 Discovery 产出物

5 份独立 Markdown 文档：
- `01-客户画像.md` —— 客户基本盘 + 业务结构 + 技术现状 + 战略意图 + 决策链
- `02-外部环境.md` —— 政策窗口 + 行业现状 + 合规要求
- `03-竞争格局.md` —— 直接对手 + 替代品 + 上游基础设施 · **必须分层（基座/行业/应用）**
- `04-学术前沿地图.md` —— 5 大主题 × 各 5-10 篇代表论文 + 工业级落地信号
- `05-跨行业 AI 实践.md` —— 至少 5 个跨行业案例 + 失败案例 + 启示

**质量门槛**：每份文档结尾必须有"未确认信息清单"列出本轮无法答的问题。

## 4 · Mode P · Planning 分析筛选与定向

### 4.1 机会池构建

从 Discovery 五份文档中提取**所有可能的 AI 应用方向**（10-20 个），不预先筛选。每个方向最少含：
- 名称 + 一句话定义
- 客户为什么需要它（痛点对位）
- 技术可行性（学术 + 工业级落地证据）
- 大致预算量级（万元级 / 百万元级 / 千万元级）

### 4.2 五把筛子（强烈推荐 · 实战沉淀）

10-20 个方向用五把筛子打分（每筛 1-5 分），过 18 分的进入 Top 名单：

| 筛子 | 评分依据 |
|---|---|
| **政策对齐度** | 是否对位现行政策窗口 / 考核节点 |
| **客户禀赋匹配** | 是否落在客户已有资质 / 资产 / 数据池上 |
| **技术成熟度** | 学术上 SOTA 是否已超过商业化阈值 |
| **市场验证度** | 跨行业是否已有可识别的成功落地案例 |
| **竞争空间** | X 层（应用）是否还未被头部完整覆盖 |

### 4.3 价值承接映射（每个 Top 方向必填）

对每个 Top 方向，必须能回答四问 —— **过不了四问的方向直接砍掉**：

1. **解决什么问题** —— 客户具体痛点（量化）
2. **提效多少** —— 工时节约 / 成本降低 / 收入增长（带【事实/推断/假设】标签）
3. **扩什么市场** —— 内部场景 → 外部产品的演进路径
4. **加速什么客户体验** —— 客户体验链条压缩一个数量级（周→天→分钟→秒）

### 4.4 三层价值承接 + L4 软实力

每个 Top 方向必须落在 L1-L4 之一，**多于一层加分**：

- **L1 内部提效** —— 1 年内可验证（直接 ROI）
- **L2 业务增长** —— 1-3 年（外部签约现金流）
- **L3 硬资产** —— 长期护城河（数据库 / 模型 / 平台）
- **L4 软实力** —— 半量化不折现（员工满意度 / 品牌地位 / 客户体验 / 组织能力 / 政策对接）

落不到三层 = 删掉。

### 4.5 Planning 产出物

- `机会池筛选.md` —— 12 方向打分表 + Top 4 详细 + 落选原因
- `价值承接映射表.md` —— Top 方向四问全过 + L1-L4 承接
- `做与不做.md` —— **关键产出**：明确列出"不做的 N 件事 + 理由 + 替代方案"，咨询价值往往在"清晰说不"
- `战略主线骨架.md` —— 产品层 + 能力层 + 业务层 + 生态层四层架构

**详细方法见 [references/02-frameworks.md](references/02-frameworks.md)**（含 BCG 10-20-70 / McKinsey SSI / 五把筛子 / 价值承接 / 战略判断分层表达 / 精益启动决策树）

### 5.2 行文严格纪律（写之前必读）

**详细见 [references/06-language-discipline.md](references/06-language-discipline.md)**

四类错误零容忍：

| 类型 | 表现 | 后果 |
|---|---|---|
| **AI 腔** | 赋能/打造/闭环/抓手/底层逻辑/颗粒度 + 综上所述/不言而喻 + "在这个 X 的时代" + 排比三连 | 整段返工 |
| **威胁恐吓** | "不投资 = 损失 X 万" / "不上车就落后" / "机会成本 X 千万" / "做比不做风险更低" | 直接删除该段 |
| **自说自话** | 把项目组自己的活动 / 项目组成员的分享 / 项目组开发的工具当外部论据 | 整段重写 |
| **跨客户内容入稿** | 把 A 客户的内部工具 / 已验证成果写进 B 客户报告（含具名工具如"X 公司的 Y skill"） | P0 必删 + 改匿名 |

**三标签纪律**（每个重要判断必标）：
- 【事实】 —— 有公开数据/文件/案例支撑
- 【推断】 —— 基于事实的合理外推，写明推理链
- 【假设】 —— 需客户内部验证，写明触发条件

## 6 · 必备纪律（写之前 + 写之中 + 写完 三个时点都要检查）

### 6.0 审稿四层认知架构(写完必跑 · 强烈建议先读)

**为什么需要四层而不只是清单**:跑完 51 项清单、跑完 6 步法,稿件仍然可能在外审被打回。原因不是清单不细,是问题永远不在第一眼看到它的层面。清单是"做什么",四层是"为什么这么做"。

| 层 | 审稿核心提问 | 典型工艺工具 |
|---|---|---|
| **L1 病因层** | 这一段在治症状还是触根因?同类反模式第二次出现 = 该停下来重新诊断 | 反模式重犯计数 / 反模式分级 |
| **L2 前提层** | 这个结论依赖的前提是否被验证?换前提结论是否还成立? | 反事实检验 + 三标签 + 对抗式自审 |
| **L3 全局层** | 各章/各表/各数字之间还一致吗?核心术语在不同章节定义有没有漂? | 单一事实源 + 概念审计 + 主稿改后回扫 |
| **L4 结构层** | 这条纪律靠人记还是靠脚本/流程拦?反复犯的失误能否下沉到结构? | 约束三级:口头 < 流程 < 脚本 |

**详细四层方法论 + 审稿顺序 + 工艺工具映射,见 [references/11-review-thinking-model.md](references/11-review-thinking-model.md)**

### 6.1 八条铁律（写之前 + 写之中）

| # | 铁律 | 应用要点 |
|---|---|---|
| 1 | **价值驱动设计** | 每个技术决策过四问（解决什么/提效多少/扩什么市场/加速什么客户体验）|
| 2 | **三标签** | 每个重要判断必标【事实】/【推断】/【假设】|
| 3 | **做与不做** | 必须有"不做什么 + 理由 + 替代"清单 |
| 4 | **三层价值承接 + L4** | L1 提效 + L2 增长 + L3 硬资产 + L4 软实力 |
| 5 | **分层表达** | 竞争分析至少分基座/行业/应用三层；避免"路径被堵死"偷懒表达 |
| 6 | **认知诚实** | 没数据不说好话；不编造叙事钩子；说"我不知道"比编故事强 |
| 7 | **客户体验压缩链条一个数量级** | 周→天→分钟→秒 是 AI 的核心商业模式 |
| 8 | **范围锁定** | 只做要求的事；发现范围外问题提一嘴不展开 |

### 6.2 八维 51 项审核清单（写完必跑）

**详细见 [references/07-review-checklist.md](references/07-review-checklist.md)**

八维：事实准确 / 逻辑严谨 / 价值承接 / 不做什么 / 术语规范 / 反 AI 腔 / 字数密度 / 客户可读

### 6.3 咨询审稿 6 步法（codex 外审 + 实战沉淀）

```
1. 先审推导链不审文笔（论据→结论的链条断没断）
2. 同层级证据支撑同层级结论（外部事实 / 项目经验 / 假设不混层）
3. 单一事实源（核心数字只能有一个出处，历史底稿降级或删除）
4. 改"客观约束 / 顺延影响"（避免威胁恐吓）
5. 高管稿结论先行（不靠"最后揭示答案"）
6. 主逻辑重写后回扫入口文件和底稿（最容易翻车的纪律 · 必须做）
```

### 6.4 反模式速查（写之前 + 写完检查）

**详细 13+ 反模式见 [references/04-anti-patterns.md](references/04-anti-patterns.md)**

最高频踩坑 Top 5（实战沉淀）：
1. 编造股权链/组织关系叙事（未经官方披露的"母集团/祖集团/牵头单位"叙事）
2. 跨客户内容入稿（把 A 客户工具/数据写进 B 客户报告）
3. 自证式外部论据（项目组自己的活动 / 分享当外部验证）
4. 引文越级（把单一来源的方法论上升为"行业必做"）
5. 活口径残留（同一文件同时说"已作废"又说"正式对外"）

## 四、关键页画面构想（5 张最难做的页）

### 第 2 页 · 一页纸结论（最重要 · 30 秒讲完）

**4 象限信息卡布局**：

```
┌────────────────────┬────────────────────┐
│ 战略建议             │ 三项约束             │
│ 启动 [战略主线]       │ ◯ [政策]            │
│ [起步时间]           │ ◯ [竞争]            │
│                    │ ◯ [禀赋]            │
├────────────────────┼────────────────────┤
│ 投入回报             │ 第一步               │
│ 三年 [X] 万          │ MVP 6-10 周         │
│ ROI [N1]/[N2]%      │ [N] 万 + [M] 人     │
│ 回本 [N3]/[N4] 月    │ 通过后再启动产品化     │
└────────────────────┴────────────────────┘
```

### 第 5 页 · 战略主线全景（整张架构总图）

**架构总图四层**：
- **顶层 · 产品**：[产品 1]（蓝） + [产品 2]（橙），两个色块明确分开
- **中层 · 能力**：共享底座 L1-L6（灰底白字），下方特化子层（橙线框）
- **下层 · 业务**：V 形聚焦三业务 + 锚点专业
- **底层 · 生态**：MCP 接口 / 硬件合作 / 可插拔接口

### 第 10 页 · 投入回报（除一页纸外最关键的一页）

**对照矩阵布局**：

```
        │ Year 1 │ Year 2 │ Year 3 │ 三年合计
────────┼────────┼────────┼────────┼─────────
保守 [X] │  ...  │  ...  │  ...  │   [X]
基准 [X] │  ...  │  ...  │  ...  │   [X]  ◀ 主推档
激进 [X] │  ...  │  ...  │  ...  │   [X]
────────┴────────┴────────┴────────┴─────────
                  ↓ 期回报
┌──────────────────────────────────────────────┐
│ L1 内部提效  [X] 万（机会成本，三年累计）         │
│ L2 业务增长  [Y] 万（外部签约现金流）            │
│ L3 硬资产    [定性描述：本体库 + 模型 + 平台]    │
│ L4 软实力    [五维半量化]                      │
└──────────────────────────────────────────────┘

        现金 ROI [N1]%  ·  综合 ROI [N2]%
        回本 [N3] 个月（现金）/ [N4] 个月（综合）
```

**讲解纪律**：基准档为主口径，**不让董事长在三档之间选**。保守和激进作"风险/上行"侧线。

### 第 16 页 · L4 软实力（决策者最关心但 ROI 算不出的一页）

**五维雷达图**：

```
                员工满意度
                    ●
                /        \
       政策对接 ●            ● 品牌地位
                \           /
                 \   3 年后  /
                  \   ↗   /
                   \  /
        组织能力 ●——————— ● 客户体验
                    ●
```

**讲解话术**：
> "ROI 算不出的部分在这里。董事长您关心的'三年后公司是什么样的'——这五维就是答案。五维都在 3 年内可观测到信号。"

### 第 17 页 · 启动前三项决策（行动页 · call to action）

**决策清单卡布局**：

| 决策项 | 内容 | 金额 | 时间窗 | 授权对象 | 风险敞口 |
|---|---|---|---|---|---|
| **决策 1** | 批准 MVP 启动 | [N] 万外部 + 内部 [M] 人抽调 | [起步月] | 战略部 + [业务部] | ≤ [N] 万，可控 |
| **决策 2** | 批准 [核心招聘] | 招聘授权（薪酬走客户既有制度） | [N] 个月内 | 人力资源 + 董事长 | 招聘成本 |
| **决策 3** | 批准 Year 1 [X] 万投入框架 | 按里程碑分批拨付（M3/M6/M9/M12 四档） | M3 第一档拨付 | 财务 + 战略部 | 单档不超 [X] 万 |

**讲解话术**：
> "我不是请您今天就批 [X] 万。我请您批的是三件事：第一，MVP 启动（[N] 万兜底）；第二，[核心招聘]；第三，Year 1 框架——按里程碑分批，每档不超 [X] 万，节点过不了下一档不拨。这样最坏情况是 [N] 万 MVP 不通过，决策风险被切到了最小。"

## 六、5 个最容易翻车的点

1. **第 2 页讲不清就完蛋**——30 秒讲不清核心建议 + 投入 + 第一步，后面再讲什么都救不回来
2. **三档 ROI 不能并列展示**——并列等于让董事长选最低档。选定基准档为主口径
3. **数据没有出处会被怼**——所有数字脚注【事实/推断/假设】+ 来源
4. **不能省略"不做什么"**——高管特别在意"边界感"
5. **决策页要具体**——不要"请董事长指示"虚收尾，要具体到金额 + 时间 + 授权

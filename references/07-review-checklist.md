## 2 · 八维 51 项清单

### 维度 1 · 事实准确（10 项）

- [ ] 1.1 所有数字标【事实/推断/假设】标签
- [ ] 1.2 政策文件标准确发布日期 + 文件号
- [ ] 1.3 客户公开数据有官方来源（年报 / 招股书 / 政府公示）
- [ ] 1.4 行业数据带来源（艾瑞 / 赛迪 / IDC / Gartner / 协会）
- [ ] 1.5 学术论文标会议名 + 年份 + 第一作者
- [ ] 1.6 跨行业案例标公司名 + 来源 URL
- [ ] 1.7 国际公司引文可在官方文档核到（不编原文）
- [ ] 1.8 客户内部数据标【需客户验证】
- [ ] 1.9 没有具名编造（无来源的精确百分比 / 无来源的客户名）
- [ ] 1.10 没有编造组织关系叙事（祖孙股权链 / 牵头单位等）

### 维度 2 · 逻辑严谨（8 项）

- [ ] 2.1 SCQA 在 §1 完整呈现
- [ ] 2.2 每章首段有该章核心结论（金字塔顶）
- [ ] 2.3 推导链可追：论据 → 中间结论 → 最终结论
- [ ] 2.4 同层级证据支撑同层级结论（不混层）
- [ ] 2.5 战略判断分至少三层（基座/行业/应用）
- [ ] 2.6 没有"路径被堵死"等粗化表达
- [ ] 2.7 数量声明与实际数量一致（标题"三个原因" → 列了 3 项）
- [ ] 2.8 跨章节引用无矛盾（§6 ROI 数字 = 附件 A1 数字）

### 维度 3 · 价值承接（6 项）

- [ ] 3.1 每个战略建议过价值四问（解决什么 / 提效多少 / 扩什么市场 / 加速什么客户体验）
- [ ] 3.2 每个战略建议落在 L1-L4 之一（多于一层加分）
- [ ] 3.3 L1 内部提效有量化（即使是【推断】也要量化）
- [ ] 3.4 L2 业务增长有客户漏斗（触达 → 意向 → 试用 → 签约）
- [ ] 3.5 L3 硬资产有定性描述（不算具体估值）
- [ ] 3.6 L4 软实力五维齐全（员工/品牌/客户/组织/政策）

### 维度 4 · 不做什么（5 项）

- [ ] 4.1 §8 应用边界 ≥ 8 项
- [ ] 4.2 每条不做带理由
- [ ] 4.3 每条不做带替代方案
- [ ] 4.4 退出条件可观测（量化指标）
- [ ] 4.5 红线表清晰（如：自研基座大模型 / Revit 通用 AI / 施工机器人 ...）

### 维度 5 · 术语规范（6 项）

- [ ] 5.1 术语首次出现用"中文（英文，缩写）"格式
- [ ] 5.2 同一概念全文统一术语，无同义词漂移
- [ ] 5.3 行业专有术语在第 0 章列出
- [ ] 5.4 缩写表完整（每个出现 2+ 次的缩写都有定义）
- [ ] 5.5 中英文混排空格规范
- [ ] 5.6 公文章节编号统一（一/二/三 + （一）（二）（三） + 1./2./3.）

### 维度 6 · 反 AI 腔（8 项）

- [ ] 6.1 高风险词（赋能/打造/闭环/抓手）零出现 或 极少（< 3 处全文）
- [ ] 6.2 套话过渡（综上所述/由此可见）零出现
- [ ] 6.3 排比三连零出现
- [ ] 6.4 设问开头零出现
- [ ] 6.5 没有"在 X 时代"修辞开头
- [ ] 6.6 没有形容词堆砌
- [ ] 6.7 没有威胁恐吓（机会成本叙事）
- [ ] 6.8 ai_slop_check.py 命中 < 5 处

### 维度 7 · 字数密度（5 项）

- [ ] 7.1 每句有新信息（删掉不影响理解的句子已删）
- [ ] 7.2 句子长度合理（多数 15-25 字）
- [ ] 7.3 段落长度合理（多数 100-200 字）
- [ ] 7.4 表格 + 散文搭配，不靠纯表格
- [ ] 7.5 总字数符合骨架（见 03-report-architecture.md）

### 维度 8 · 客户可读（3 项）

- [ ] 8.1 非技术读者也能读懂 §1 执行摘要
- [ ] 8.2 §10 决策点明确具体（金额 / 时间窗 / 授权对象 / 风险敞口）
- [ ] 8.3 一页纸版本可独立阅读（30 秒讲完核心建议）

## 3 · 咨询审稿 6 步法

> 沉淀自 codex 多轮外审 + 实战总结。每步必跑，**第 6 步是最关键的纪律**。

### 步骤 1 · 先审推导链不审文笔

**目标**：判断每个核心结论的论据 → 结论链条是否完整。

**操作**：
- 列出报告中所有"核心结论"（一般在每章开头或 §1）
- 对每个结论，向上追溯支撑论据
- 检查论据 → 中间结论 → 最终结论的每一步是否成立

**典型问题**：
- 结论与论据不匹配（结论说"必须做"，论据只支持"可以做"）
- 中间结论跳跃（A → C 之间缺 B）
- 论据互斥（论据 1 与论据 2 实际矛盾）

### 步骤 2 · 同层级证据支撑同层级结论

**目标**：避免"事实 / 经验 / 假设"混层支撑结论。

**操作**：
- 检查每个结论的证据来源标签
- 高层级结论（战略选择 / 重大决策）必须用【事实】+【推断】支撑
- 不允许只用【假设】支撑高层级结论

**典型问题**：
- 战略主线选择只靠【假设】（"客户可能想做 X"）
- 客户禀赋分析混入项目组经验

### 步骤 3 · 单一事实源

**目标**：核心数字只能有一个出处。

**操作**：
- 列出所有核心数字（投入 / 回报 / ROI / 客户单价 / 工时节约）
- 检查这些数字在不同文档中是否一致
- 如有历史口径，标【已作废】并明示新口径位置

**典型问题**：
- 主报告 §6 写新数字，附件 A1 写旧数字
- 同一文件同时说"已作废"+"正式对外"
- 跨文档引用链断裂

### 步骤 4 · 改"客观约束 / 顺延影响"

**目标**：用客观叙事替代威胁恐吓。

**操作**：
- grep 所有威胁恐吓语句（"不做 = 损失"）
- 改为客观陈述（"X 情况下，Y 业务面临 Z 影响"）
- 让客户自己得出"不做的代价"

### 步骤 5 · 高管稿结论先行

**目标**：避免"最后揭示答案"式叙事，确保结论在前。

**操作**：
- 检查 §1 是否在前 200 字内给出核心建议
- 检查每章是否在首段给出该章结论
- 一页纸是否在 30 秒内讲清"建议什么 / 投入多少 / 第一步"

### 步骤 6 · 主逻辑重写后回扫入口文件和底稿（最关键）

**目标**：避免"主稿改了，底稿没改"导致的口径不一致。

**触发条件**：主稿 §3 战略主线 / §6 ROI / §9 组织 任何一处改动。

**操作**：
- 列出所有引用主稿的文档（附件 / 一页纸 / PPT / 其他章节 / CLAUDE.md / 项目档案）
- 逐一回扫，确保数字 / 表述 / 决策一致
- 历史口径标【已作废】并明示新口径位置

**实战教训**：4 条 P 级问题（占位文件号 / ROI 三套数字 / 人员曲线打架 / 第 0 章错位）全部源于"重写主文后没回扫源文档"。

## 3.5 · 对抗式自审（Adversarial Self-Review · 主关键结论必跑）

> 6 步法之后,**每个核心战略建议**还要再过一道对抗式自审。这是把"我说服自己"切换为"假装外审反驳我"的最后一道关卡。

### 操作

对每个核心结论(战略主线 / 投入档位 / 第一步建议),执行 3 步:

1. **写一个最强反方意见**(用一句话):"如果一个反对者要反驳本结论,他最有力的论据是什么?"
2. **给本结论评级**(5 档置信度):
   - 【事实】 —— 已有公开数据或客户内部证据可验
   - 【强推断】 —— 多源信号相互印证
   - 【弱推断】 —— 仅 1-2 条信号支撑
   - 【假设】 —— 需客户内部数据验证
   - 【未知】 —— 信息不足,该结论应改为"待评估"或暂缓
3. **任何评为【弱推断】或更低的结论**必须加 [需要客户验证] 标注,并在 §10 启动建议里列出需要的验证动作

### 示例

| 核心结论 | 反方意见 | 我的评级 | 处理 |
|---|---|---|---|
| 在 2026 上半年启动战略 | "政策窗口关闭时间是 2027 末,可以再观察 6 个月" | 【强推断】 | 保留;在 §2 政策环境里答复反方 |
| 三年综合 ROI 339% | "L2 客户漏斗依赖项目组销售能力,客户自身销售团队是否承接是未知" | 【弱推断】 | 加 [需客户销售团队评估] 标 + §6.5 敏感性分析展示 -50% 场景 |
| 自研基座大模型 | "国内已有 Qwen/DeepSeek 等开源基座,自研是否资源浪费" | 【假设】 | 改为"基于开源基座做行业 LoRA 微调",不写"自研基座" |

**纪律**:对抗式自审做完才算 §1 执行摘要可定稿。任何一个核心结论评级低于【强推断】而没有标注 [需要验证] = 当稿件给客户读时一定会被反问。

## 4 · 外审触发条件

下列情形触发 codex 或同等强度外审：

| 触发情形 | 必要性 |
|---|---|
| 客户终审前 | 必须 |
| 主稿做过 ≥ 2 轮重大修订 | 必须 |
| 战略判断有重大变化（如方案 A → 方案 B 切换） | 必须 |
| 客户付费规模 > 100 万 | 强烈建议 |
| 报告涉及客户敏感事项（股权 / 政策对接 / 重大投资） | 强烈建议 |
| 项目组成员对方案有分歧 | 建议 |

**外审纪律**：
- 外审必须独立于主稿撰写者
- 外审反馈分 P0 / P1 / P2 / P3 四级
- P0 必修；P1 必改；P2/P3 视情况
- 修订后必须再 grep 验证（不能只信主笔说"改了"）

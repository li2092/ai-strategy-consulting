## 1 · ROI 路线选择决策树

```
客户单项目 IT 投入预算（年）
├── < 100 万 → 精益启动路线（小团队 + MVP + 渐进扩展）
│   适用：国企工程基建 / 中小政府客户 / 行业咨询型企业
│   团队规模：Year 3 稳定 5-7 专职 + 4-5 兼职
│   三年总投入：400-520 万
│
├── 100-500 万 → 标准路线（中等规模团队 + 模块化产品）
│   适用：金融中型机构 / 制造业上市公司 / 科技公司
│   团队规模：Year 3 稳定 10-15 专职
│   三年总投入：1500-3000 万
│
└── > 500 万 → 重资产路线（豪华团队 + 全功能平台）
    适用：央企 IT / 大型政府平台 / 头部金融
    团队规模：Year 3 稳定 20-50 专职
    三年总投入：5000-15000 万
```

**判断纪律**：
- 客户付费能力决定路线（不是项目雄心决定路线）
- 国际平台型公司的高单价案例只能借鉴方法，不能直接迁移经济模型到中小预算客户身上
- 误用路线 = 提案被否（客户付不起 / 客户认为低估）

## 2 · 精益启动 ROI 主模板（推荐）

**典型客户**：国企工程基建 / 中小政府 / 行业咨询型 / 中型 ISV

**三档投入**：
| 档位 | 三年总投入 | 年均投入 | 触发条件 |
|---|---|---|---|
| **保守** | 480-560 万 | 160-187 万 | AI 协同度不达预期，团队学习曲线慢 |
| **基准** | 420-500 万 | 140-167 万 | AI 协同达预期（**主推口径**） |
| **激进** | 360-440 万 | 120-147 万 | AI 协同度高，代码工程占比高 |

**投入分类**（基准档结构示例 · 数字按客户实际付费能力替换）：
| 类别 | Year 1 | Year 2 | Year 3 | 三年合计 | 占比参考 |
|---|---|---|---|---|---|
| 自研小模型外部成本 | a | a | a | 3a | 10-15% |
| 内部团队人力（折算）| b1 | b2 | b3 | Σb | 55-65% |
| 数据治理 + 规范库 | c1 | c2 | c3 | Σc | 7-10% |
| 客户推广 + 培训 | d1 | d2 | d3 | Σd | 12-18% |
| 杂项（差旅 / 评测 / 工具） | e1 | e2 | e3 | Σe | 3-5% |
| **合计** | **Y1** | **Y2** | **Y3** | **总投入** | **100%** |

**三年回报**（基准档结构示例 · 数字按客户实际签约能力替换）：
| 层 | 类别 | Year 1 | Year 2 | Year 3 | 三年合计 |
|---|---|---|---|---|---|
| L1 | 内部提效（机会成本） | 见 §5 | | | Σ L1 |
| L2 | 业务增长（外部签约） | 见 §6 | | | Σ L2 |
| L3 | 硬资产（不折现） | — | — | — | 平台 + 模型 + 数据 |
| L4 | 软实力（半量化） | — | — | — | 五维信号 |

**综合 ROI 公式**：
- 现金 ROI = Σ L2 / 三年总投入 ≈ **N1 %**
- 综合 ROI = (Σ L1 + Σ L2) / 三年总投入 ≈ **N2 %**
- 现金回本期 ≈ **N3 个月**
- 综合回本期 ≈ **N4 个月**

**数字范围参考（仅作量级校准用，不直接套用）**：
- 精益启动客户的综合 ROI 通常落在 200-400% 之间，超出区间须重新核对锚点
- 现金回本期通常 18-30 个月；综合回本期通常 12-18 个月
- 若数字明显落出区间，先怀疑客户单价 / 工时单价 / 落地率假设是否过激

## 5 · L1 内部提效计算

### 5.1 计算公式

```
L1 三年合计 = Σ (年节约工时 × 工时单价 × 已落地业务比例)
```

### 5.2 关键参数

| 参数 | 取值范围 | 来源 |
|---|---|---|
| 单业务工程师数 | 视客户内部数据 | 客户提供 |
| 单工程师年工时 | 1900-2200 小时（中国法定）| 公开统计 |
| 工时单价 | 80-300 元/小时（视职级 + 行业）| 客户提供 / 行业平均 |
| AI 加持后工时节约率 | 10-30%（视场景）| 学术 / 工业级落地参考 |
| 落地业务比例 | Year 1: 20-30% / Year 2: 50-60% / Year 3: 80-100% | 推断 |

### 5.3 标签纪律

L1 测算几乎都是【推断】（基于学术指标 + 行业类比），少数是【假设】（如客户具体的工时节约率）。**禁止标【事实】**。

### 5.4 通用计算示例

```
某客户 · 聚焦 N 条业务线工程师共 X 人 · 平均工时单价 Y 元 / 小时
- Year 1：X × 2000 × Y × 15% × 25% ≈ L1_y1 万
- Year 2：X × 2000 × Y × 18% × 55% ≈ L1_y2 万
- Year 3：X × 2000 × Y × 20% × 90% ≈ L1_y3 万
- 三年合计 L1 ≈ Σ L1 万【推断】
```

**参数敏感度提示**：
- AI 加持后工时节约率（15-30%）是最关键变量，必须基于客户具体场景做 MVP 验证后回归
- 落地业务比例（Year 1: 20-30%）取保守值更稳；激进值会被外审反推

## 6 · L2 业务增长计算（客户漏斗）

### 6.1 三档客户单价

按行业适配（见 §1 决策树），取定档位。

### 6.2 三年签约漏斗

| 阶段 | Year 1 | Year 2 | Year 3 |
|---|---|---|---|
| **触达客户数** | 内部 + 5-10 | 30-50 | 50-100 |
| **意向客户数** | 3-5 | 10-15 | 20-30 |
| **试用客户数** | 1-3 | 5-10 | 10-20 |
| **签约客户数** | 0-2 | 3-8 | 8-15 |
| **基础档** | 0-1 | 3-5 | 5-8 |
| **进阶档** | 0 | 1-2 | 2-4 |
| **旗舰档** | 0 | 0-1 | 1-3 |

### 6.3 L2 计算公式

```
L2 年度 = Σ (各档客户数 × 各档单价)
```

### 6.4 通用计算示例

某工程基建 / 咨询监理类客户（精益启动）：
```
Year 1：1 基础档（15 万）= 15 万
Year 2：3 基础 + 1 进阶 + 0 旗舰 = 3×15 + 1×40 = 85 万
Year 3：5 基础 + 2 进阶 + 1 旗舰 = 5×15 + 2×40 + 1×100 = 255 万
（三档实际签约通常 50% 实现概率，需按客户销售团队规模回归）
三年合计 L2 中性约 N 百万级 · 视 conversion rate【推断 + 假设】
```

**反模式提示**：直接把单价档位乘以理论最大签约数得出"最佳情况"数字会被外审判定夸大，必须按客户实际触达 / 转化漏斗回归。

### 6.5 标签纪律

L2 几乎全部【假设】（客户漏斗 + 转化率 + 单价均依赖客户内部数据）。**写明触发条件**：客户实际付费能力 / 销售团队规模 / 市场推广力度。

## 7 · L3 硬资产估值

### 7.1 计算原则

L3 不直接折现到 ROI，但作为"长期护城河"独立列出。

### 7.2 典型 L3 资产

| 资产 | 估值依据 |
|---|---|
| 行业知识图谱 / 规范库 | 重置成本（从零建设需要的人力 × 时间） |
| 自研小模型 | LoRA 训练成本 + 数据标注成本 + 调优工时 |
| 行业 Agent 平台 | 平台开发工时 + 集成接口 + 可复用模块数 |
| 数据资产（决策回流库） | 业务决策记录数 × 单条价值（参考行业 LLM 数据集定价） |

### 7.3 不写量化估值

**不要算"L3 资产值 X 千万"**——容易被判定夸大，且无标准估值方法。

**改为定性表达**：
> "Year 3 末客户拥有：完整行业规范知识图谱（覆盖 N 本核心规范）、自研行业小模型（基于 Qwen3-14B LoRA 微调）、决策回流库（累计 N 条业务判断记录）。这些资产构成离开本平台无法快速重建的护城河。"

## 8 · L4 软实力五维表达

### 8.1 五维结构

| 维度 | 度量信号 | 时间窗 |
|---|---|---|
| **员工满意度** | AI 工具加持 → 重复劳动减少 → 高价值工作占比 | 半年期看变化，年度看趋势 |
| **品牌地位** | 行业大赛入选 / 政策项目落地 / 媒体能见度 | 季度看信号 |
| **客户体验** | 响应链条压缩 → 客户粘性 / 续约率 | 半年期看 |
| **组织能力** | 知识沉淀 → 离职率风险 ↓ + 新人上手期 ↓ | 年度看 |
| **政策对接** | 进入部委名单 / 政策红利获取 | 不定期 |

### 8.2 表达模板

```markdown
## L4 软实力（半量化 · 不折现）

ROI 算不出的部分体现在五个维度。这五维的信号在 3 年内可观测：

| 维度 | 当前基线 | 3 年后预期 | 度量方式 |
|---|---|---|---|
| 员工满意度 | [当前 X 分 / 100] | [预期 +N 分] | 年度员工调研（净推荐值） |
| 品牌地位 | [当前 X 名] | [预期进入 Y 名] | 行业排名（如建筑业百强）/ 媒体提及次数 |
| 客户体验 | [当前续约率 X%] | [预期 +N pp] | 客户满意度调研 / 续约率 |
| 组织能力 | [当前离职率 X%] | [预期 -N pp] | HR 数据 / 新人上手期 |
| 政策对接 | [当前已对接 X 个部委] | [预期进入 Y 名单] | 政府文件 / 项目入选 |

**为什么必须独立列出 L4**：
决策者关心"事成后公司是什么样"，不只看 ROI。L4 在 ROI 算不出的部分提供"软回报"叙事。
```

### 8.3 不算入综合 ROI

**禁止**把 L4 折现成"软实力价值 X 千万"——这种算法会被判 P0 编造数据。

**正确做法**：把 L4 列在 ROI 表后单独成节，让客户自行体会"事成后公司是什么样"。

## 9 · 综合 ROI 与回本期

### 9.1 公式

```
现金 ROI = L2 三年合计 / 三年总投入
综合 ROI = (L1 + L2) 三年合计 / 三年总投入
现金回本期 = 三年总投入 / (L2 年均现金流)
综合回本期 = 三年总投入 / ((L1 + L2) 年均现金流)
```

### 9.2 表达纪律

- 必须同时给"现金 ROI"和"综合 ROI"两个口径
- 现金 ROI 是底线（保守投资人看），综合 ROI 是上行（含 L1 机会成本）
- 回本期同样两个口径
- 三档（保守 / 基准 / 激进）只主推**基准档**，不并列三档让客户挑

### 9.3 一句话口径模板

```
基准档三年投入约 X 万元（年均 Y 万元），L1+L2 三年合计回报约 Z 万元，
现金 ROI 约 N1%，综合 ROI 约 N2%，回本期 N3 个月（现金）/ N4 个月（综合）。
保守档即便 L2 仅实现 50%，综合 ROI 仍有约 N5%——
L1 内部提效的可控性构成决策的下限保障。
```

## 10 · 敏感性分析与对外口径

### 10.1 敏感性变量

| 变量 | 浮动范围 | 影响 |
|---|---|---|
| L2 客户单价 | -30% / +30% | L2 / 综合 ROI 直接同比例变 |
| L2 签约客户数 | -50% / +50% | L2 同比例变 |
| L1 工时节约率 | -40% / +40% | L1 同比例变 |
| 投入端 AI 加持效率 | -30% / +30% | 投入端反向 |
| Year 1 起步延迟 | 0-6 个月 | 综合回本期 +N 个月 |

### 10.2 敏感性表达模板

```
敏感性分析（基准档为锚点 · 数字按客户实测替换）：
- L2 单价 -30%：综合 ROI 由 N% 降至 N1%
- L2 签约 -50%：综合 ROI 由 N% 降至 N2%
- L1 节约 -40%：综合 ROI 由 N% 降至 N3%
最坏组合（同时三降）：综合 ROI 是否仍正向回报，回本期延至 X 个月
```

**敏感性表达纪律**：
- 三档敏感性必须同时给（单变量 × 多变量），不允许只展示对客户有利的子集
- 最坏组合必须计算并写出，避免"只展示乐观情景"

### 10.3 对外口径纪律

- **不并列三档**让客户自选（会选最低档）
- 选定基准档为主口径
- 保守档作"风险下界"，激进档作"上行潜力"
- 回本期用"综合"和"现金"两个口径同时给
- 禁止"机会成本威胁"叙事
- 禁止"AI 溢价"假设

# 战略深度 · 5 项硬约束详细标准

> 这份文档解决一个核心矛盾:战略报告**止步于"为什么/做什么",缺"怎么做"的最低限度具象化**——所以读者会觉得"MCP 方案比战略报告还具体"。
>
> 5 项硬约束(H1-H5)就是把这层窗户纸捅破:不允许停在抽象层,必须落到可视化锚点 + 具体对标 + 明确取舍。

---

## 0 · 5 项约束总览

| # | 约束 | 适用 | 解决的矛盾 |
|---|------|------|----------|
| H1 | 表格字数 ≤ 25% | 每节/每章 | 模板表格化 → AI 模仿写成全表格(没论证、没人味) |
| H2 | 每节 ≥ 2 段散文(150-300 字/段) | 每节/每章 | 同上 + 强制论证连贯性 |
| H3 | ≥ 1 个具象化锚点(输入 → 处理 → 输出) | 每个 Top 推荐 | 战略止步于"为什么/做什么",缺"怎么做"的可视化 |
| H4 | ≥ 1 个外部对标(具名公司+产品+时间+规模) | 每个 Top 推荐 | "某某科技巨头在做"等于没说 |
| H5 | ≥ 1 个取舍段(为什么不做 X) | 每个 Top 推荐 | 只推荐不取舍 = 没有战略判断 |

---

## 1 · H1 · 表格字数 ≤ 25%

### 1.1 为什么这条

读者打开战略报告希望看到**判断和论证**,不是数据汇编。表格用来对比、列清单、做摘要,不能承载战略判断。判断必须用散文,因为判断有"我认为...所以...因此..."的逻辑链,这种链条用表格切碎后就不是判断了,只是数据陈列。

实战观察:AI 默认走"先列表 + 后注释"路径,因为表格语法比段落短、产出快。但读者收到 30 个表格的报告会立刻判断"这是 AI 写的,没人审过"。

### 1.2 自查方法

```
表格字数 ≈ 表格行数 × 每行平均字数(粗估即可)
表格占比 = 表格字数 / 全章总字数
```

举例:一章 3000 字,3 个表格各 8 行,每行约 30 字 = 720 字,占比 24%,刚好达标。

### 1.3 好 / 差对照

**差**:
```
本章分析 Top 3 推荐方向。

| 方向 | 痛点 | 技术 | 提效 | 风险 |
|------|------|------|------|------|
| A | XX | RAG | 30% | 中 |
| B | YY | Agent | 50% | 高 |
| C | ZZ | LoRA | 20% | 低 |

(然后没了,或者每个方向再 1-2 句话)
```

**好**:
```
本章分析 Top 3 推荐方向。三个方向针对不同痛点,采用不同技术路径,
风险和提效水平也分布在三个量级。下面分别展开。

(2-3 段散文,讲清楚 A/B/C 的判断依据)

为便于横向对比,把关键参数汇总如下:

| 方向 | 痛点 | 技术 | 提效 | 风险 |
|------|------|------|------|------|
| ... |

(表格之后再 1-2 段散文,讲表格里看不出的取舍)
```

差的版本表格占比 80%+,好的版本表格占比 20%。

## 2 · H2 · 每节 ≥ 2 段散文,每段 150-300 字

### 2.1 为什么这条

强制论证连贯性。一段 150-300 字的散文必须有"主张 + 证据 + 推论"的完整逻辑;只有 50 字的散文只能陈述事实,不能论证。

### 2.2 散文段定义

不算散文段:
- 标题 / 副标题
- 表格
- 列表(`-` `*` `1.` 开头)
- 引用块(`>` 开头)
- 代码块

只算:
- 完整段落(以句号或问号结尾)
- 长度 ≥ 150 字

### 2.3 自查方法

每节写完,逐段数:满足"段长 ≥ 150 字 + 不属于以上排除项"的段数,目标 ≥ 2。

### 2.4 好 / 差对照

**差**(段太短,只是陈述,无论证):
```
本节分析腾讯广告的 AI 提效空间。

- 投放策略生成:节约策略师 50% 时间
- 创意素材生成:节约设计师 60% 时间
- 投放数据分析:节约分析师 40% 时间

综上,广告业务 AI 提效空间显著。
```

**好**(段够长,有判断有论证):
```
本节分析腾讯广告业务的 AI 提效空间。先讲个总体判断:广告业务的提效
空间不在投放本身,在投放前的策略生成和投放后的数据分析。投放本身
是高度自动化的(竞价系统已是 AI),策略和分析才是策略师/分析师每天
花大量时间的环节,这两个环节的提效空间最大。

具体看,策略师每天花 3-4 小时做客户行业研究、竞品对标、投放预算分配,
这三件事都可以由 LLM + RAG 替代 70% 工时。分析师每周 1-2 天做投放
归因报告,这部分可以由数据 Agent + 图表生成自动化,人只做最终判断。
创意素材生成对设计师的替代率反而较低,因为腾讯广告的核心客户(美妆/
游戏/电商)对素材调性要求高,AI 生成的素材目前只能做初稿,不能直接
交付。
```

差的版本 90 字,只有结论没论证;好的版本 350 字,有判断、有数据、有取舍。

## 3 · H3 · 具象化锚点(输入 → 处理 → 输出)★ 关键

### 3.1 为什么这条

这是解决"战略报告比 MCP 方案抽象"的核心。每个推荐方向必须给出一个**可视化的工作流锚点**——让读者能想象"这个方向落地后,实际的工作流大概是什么样"。

锚点不是要写技术方案(那是后续工程师的事),是给读者一个**心智模型**:输入什么数据 → AI 做什么处理 → 输出什么结果。

### 3.2 锚点格式

```
【场景名】[一句话定义]
- **输入**:[具体数据/触发条件,带量级]
- **AI 处理**:[1-3 步关键动作,具体到模型/技术路径]
- **输出**:[具体产物 + 给谁用 + 用来做什么]
- **替代了原来的**:[人/流程/工具]
- **保留人做的**:[判断/审核/边界]
```

### 3.3 好 / 差对照

**差**(泛泛、没锚点):
```
推荐方向 1:用 LinkedIn AI 提升广告投放效率。

LinkedIn 等领先公司已经在使用 AI 优化广告投放,我们也应该跟进。
预计可以提升投放效率 30%-50%。
```

读者看完无法想象这是什么——"用 AI 优化"是说什么?给广告主用还是给运营用?提升的是哪个环节?

**好**(有具象化锚点):
```
推荐方向 1:策略师 Copilot · LLM + RAG 替代客户行业研究 70% 工时

【场景】客户接入后,策略师需要 1-2 天调研客户行业、竞品、关键投放词。

- **输入**:客户名称 + 客户行业归属 + 客户给的预算 + 客户提供的过往投放数据
- **AI 处理**:
  1. RAG 检索内部行业研究库(含 2 年腾讯广告所有客户行业报告)+ 竞品库
  2. LLM 生成客户行业概览(规模/增速/Top 10 玩家/广告投放风格)
  3. LLM 提议 3 套投放策略(对标竞品 / 差异化 / 实验性)
- **输出**:一份 5-8 页"客户行业研究 + 投放策略草案"草稿,Markdown 格式,1-2 分钟生成
- **替代了原来的**:策略师 1.5 天的检索 + 撰写工作
- **保留人做的**:看草稿后基于客户对接情况做最终判断 + 与客户的方案沟通
```

读者一看就能想象——"哦,就是给策略师做了个研究助手"。这种锚点让战略报告从"概念"变成"可想象的工具"。

### 3.4 H3 的最低标准

锚点的 5 个要素至少要有 4 个具体填充:
- 输入(必须具体到数据来源 + 量级)
- AI 处理(必须有 1-3 步,不能只写"LLM 处理")
- 输出(必须具体到形式 + 用途)
- 替代了原来的(必须具体到人/流程)
- 保留人做的(必须明确,避免"AI 全自动"幻觉)

5 项缺 2 项以上 → H3 ✗。

## 4 · H4 · 外部对标(具名公司 + 产品 + 时间 + 规模或效果)★ 关键

### 4.1 为什么这条

战略报告的根基是"对标世界前沿"。如果对标只说"某科技巨头在做"、"行业领先公司已部署",等于没对标——读者会怀疑"你是不是在编"。

具名对标 = 暴露你做过功课 + 让读者可验证 + 让推荐方向有外部锚定。

### 4.2 对标格式

```
**对标**:[公司名] 在 [时间] 推出/部署 [产品/项目名],
[一句话核心做法],[一个可验证的效果数字 或 规模信息]。
来源:[文章/报告/blog/论文 链接 或 名称]。
```

### 4.3 好 / 差对照

**差**:
```
对标:LinkedIn 等领先公司已经在使用 AI 优化销售流程。
```

读者疑问:LinkedIn 哪个产品?哪一年?具体做什么?效果如何?来源是哪?

**好**:
```
对标:LinkedIn 在 2024 年 Q3 推出 "Sales Navigator AI",
对销售人员的客户研究环节做了 LLM Copilot 集成,
内部数据显示销售人员平均每天节约 1.5 小时客户调研时间,
覆盖 LinkedIn 自有销售团队约 8000 人。
来源:LinkedIn Engineering Blog 2024-10-15
"Building Generative AI for Sales Navigator"。
```

读者看完知道:有这个产品、有这个时间、有这个数字、有这个来源、可去验证。

### 4.4 H4 的最低标准

四个要素必须齐全:
1. **公司名**(具体到产品/团队层级,不能只说"某硅谷大厂")
2. **产品/项目名**(必须有具名)
3. **时间**(精确到年/季度,不能只说"近年")
4. **规模或效果**(任选一个:覆盖多少人/团队 / 提效多少 / 收入影响)

4 项缺 1 项以上 → H4 ✗。**找不到具名对标 → 这个方向不要推荐**(没对标说明该方向不成熟,推荐了也没说服力)。

### 4.5 对标的诚实纪律

- **找不到对标 ≠ 编一个**。找不到 → 在 Checkpoint 报告"H4 这条找不到合适对标,建议候选 X 不进 Top 3"
- **对标不能用项目组自己的活动**(参 §6 反模式 5,自说自话)
- **二手转述要标明**:看到"听说 OpenAI 在做..."的传闻 → 写成 "【推断·二手】LinkedIn 内部某团队在 2024 年讨论..." 而不是写成事实

## 5 · H5 · 取舍段(为什么不做 X)★ 关键

### 5.1 为什么这条

战略 = 选择 = 取舍。**只推荐不取舍 = 没做战略判断,只做了汇编**。

读者(尤其是 CTO / BU 总裁级)看战略报告时,最关心的不是"你推荐了什么",而是"你看过哪些选项 + 为什么没选其他"。看到只有推荐没有取舍的报告 → 立刻怀疑"是不是没真做思考"。

### 5.2 取舍段格式

```
**为什么不做 X**:[一句话核心理由,通常是"在客户当前阶段不划算" /
"技术不成熟" / "ROI 不达预期" / "和已有投资冲突" / "对手已建立壁垒"]。
[1-2 句展开,带数据或对标]。
```

### 5.3 好 / 差对照

**差**:
```
推荐方向 1:策略师 Copilot
推荐方向 2:分析师 Copilot
推荐方向 3:创意素材生成

(然后没有取舍段)
```

**好**:
```
推荐方向 1:策略师 Copilot
推荐方向 2:分析师 Copilot
推荐方向 3:创意素材生成

**为什么不做投放出价 AI**:腾讯广告的竞价系统已经是高度优化的 AI
(2018 年起持续迭代),内部资源不应再投入做"用大模型重写竞价"——
这是把石头扔进湖里,看不到水花。世界对标:Google Ads 也没用 LLM
重写过其竞价系统,LLM 用在策略和创意环节。

**为什么不做客户全自助 AI Agent**:让客户自己用 Agent 投广告,看似
是颠覆性方向,但腾讯广告的核心客户(美妆/游戏/电商)依赖销售/客户成功
团队的关系经营,贸然推全自助 = 砍掉销售团队 = 客户流失风险。Meta Ads
推全自助多年,头部广告主仍然走销售渠道。

**为什么不做生成式视频广告**:视频生成模型(Sora 类)2025 年仍处于
"能演示不能商用"阶段,广告主对调性把控严格,直接接入会产生大量品牌
事故。建议 6 个月后重评估。
```

3 个取舍段,读者立刻明白"你做过功课,看过这些选项,知道为什么不选"——战略判断的厚度立刻出来。

### 5.4 H5 的最低标准

每个推荐方向至少 1 个取舍段(说明"为什么这个方向之外,我们不做相关的 X")。

更好的做法:整章末尾有 1 个**集中取舍章节**(§5 不做什么),里面有 3-5 个取舍段。

取舍段缺失 → H5 ✗。

---

## 6 · 5 项约束的优先级

如果某节字数有限,优先满足顺序:

```
H5(取舍)> H4(对标)> H3(锚点)> H2(散文)> H1(表格 ≤ 25%)
```

为什么这个优先级:
- H5/H4/H3 是**战略深度**约束(决定报告"够不够战略")
- H2/H1 是**形式约束**(决定报告"读不读得下去")
- 战略深度比形式重要——一份散文+对标+取舍齐全的报告,即使表格占比 30% 也比纯表格的好

但**所有约束 ✗ 都必须在 Checkpoint 报告**,不能因为优先级低就藏着。

---

## 7 · Step 1 vs Step 2 的微调

| 约束 | Step 1 | Step 2 |
|------|--------|--------|
| H1 表格 ≤ 25% | 适用 | 适用 |
| H2 散文段 ≥ 2 | 适用 | 适用 |
| H3 锚点 | Top 3 推荐每个 ≥ 1 | 每个推荐方向章 ≥ 2 |
| H4 对标 | Top 3 推荐每个 ≥ 1 | 每个推荐方向章 ≥ 2 |
| H5 取舍 | 整个 §五 不做什么 ≥ 3 个 | 整个 §5 取舍章 ≥ 3 个,且每个推荐方向自带 ≥ 1 个 |

Step 2 比 Step 1 更严,因为正式报告读者级别更高、容忍度更低。

## 8 · 写之前的自检(每个推荐方向开写前)

写每个推荐方向前,先填这张卡:

```
【方向名】___________
【一句话定义】___________

H3 锚点:
- 输入:___________
- AI 处理:___________
- 输出:___________
- 替代了:___________
- 保留人做的:___________

H4 对标:
- 公司:___________
- 产品/项目名:___________
- 时间:___________
- 规模或效果:___________
- 来源:___________

H5 取舍:
- 不做的方向:___________
- 不做的核心理由:___________
- 一句话展开:___________
```

填不出来 → **不要开写这个方向**,先去做调研或换方向。

写之前填卡 = 避免写到一半发现"对标找不到、锚点说不清"再返工。

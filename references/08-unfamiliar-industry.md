# 陌生行业摸底框架(核心通用能力)

> **本文件回答的问题**:用户提出的行业属于陌生领域(如宠物医院 / 光伏组件回收 / 民营养老 / 二手奢侈品 / 殡葬 / 印刷 / 园林...),Step 1 行业速览没有现成样板,skill 怎么办?
>
> 答案:**框架优先 + 实例补充**。任何陌生行业都能用本框架 1-2 小时建立够用的认知,然后接入 [Step 1 探索方向](01-step1-exploration.md) → [Step 2 写战略报告](02-step2-drafting.md) 的主流程。

---

## 0 · 为什么需要"框架"而不是"穷举行业"

行业有几千个,样板库永远填不全(餐饮 / 宠物 / 殡葬 / 印刷 / 婚庆 / 民宿 / 二手车 / 园林 / 医美 / 律所 / 会计师事务所 / 私募 / 物流园区 / 跨境电商 / 直播 MCN / ...)。

两种思路对比:

| 思路 | 风险 | 评价 |
|---|---|---|
| 给每个行业写一份 checklist | 永远填不全 + 维护负担巨大 + 行业新业态出现就过时 | 错误的方向 |
| **写一个能扛任何行业的摸底框架** | 框架本身是通用的,实例可以慢慢沉淀 | **正确方向** |

这正合 [07-review-thinking-model.md L4 结构层](07-review-thinking-model.md) 的思想:把"陌生行业怎么摸底"沉淀为**结构性能力**,不依赖"每次靠人想该查啥"。

---

## 1 · 启动判断 · 这个行业在样板库吗

```
用户提出行业 X
   │
   ▼
[判断 X 是否在样板库]
   │
   ├── 在(国企央企 / 民营上市 / 创业 / 政府事业 / 医疗 / 教育 / 制造 / 物流 / 零售 / 农业)
   │   └── 直接用 [Step 1 行业速览](01-step1-exploration.md) 的常见行业知识
   │
   └── 不在(任意其他行业)
       └── 跑本框架 §4 的 5 步法 → 产出 A 行业知识背景 + 产出 B 应用框架
           然后接 [Step 1 探索方向](01-step1-exploration.md) → [Step 2 写战略报告](02-step2-drafting.md)
```

**判断纪律**:不要硬把陌生行业塞进相近样板("宠物医院 ≈ 医疗") —— 不同业态的价值链 / 客户决策链 / 政策约束差异巨大,套错样板比没样板更糟。**不在样板库就跑 5 步法,这是底线**。

## 2 · 产出物 A · 行业知识背景模板

跑完 5 步法后,**第一份产出**:`行业_X_知识背景.md`(2-3 页)。

```markdown
# 行业 X · 知识背景

## 一、基本盘
- 行业定义(50 字内 · 用一句话讲清这个行业卖什么)
- 行业规模(年营收 / 客户数 / 从业人员数,带来源)
- 增速(近 3 年 CAGR / 未来 3 年预测)
- 集中度(Top 5 占比,判断是分散市场还是寡头市场)

## 二、价值链结构
- 上游(供给侧:谁给本行业提供原料 / 服务 / 牌照)
- 中游(本行业自身:主要业态 + 收入模式)
- 下游(需求侧:谁付钱?B / G / C?客单价区间?)
- 价值集中点(哪一段毛利最高?哪一段最易被替代?)

## 三、头部 3 家速览
| 公司 | 规模 | 模式特点 | AI 投入(若有公开信息) |
|---|---|---|---|
| 龙头 1 | ... | ... | ... |
| 龙头 2 | ... | ... | ... |
| 颠覆者(若有) | ... | ... | ... |

## 四、政策与监管约束
- 行业归口部委(如有)
- 关键牌照 / 资质 / 准入门槛
- 近 18 个月重要政策
- 数字化 / AI 相关的合规要求

## 五、行业核心痛点(优先列 5 个)
1. 痛点 X1 —— 具体描述 + 受影响环节
2. 痛点 X2 —— ...
3. ...

## 六、数字化与 AI 成熟度
- 行业整体处于:信息化 / 数字化 / 智能化 哪一阶段
- 已有的 AI 应用案例(若有)
- AI 渗透率估算【推断】

## 七、未确认信息清单
- [ ] 客户私域:实际付费能力 / 决策时间窗 / 内部 IT 现状
- [ ] 行业专有约束:[需追加调研项]
- [ ] ...
```

## 4 · 5 步法详解

### 步骤 1 · 找该行业最权威的 1-2 份报告(20-30 分钟)

**目标**:在 30 分钟内对行业体量 / 增速 / 价值链有量化认知。

**优先级排序**(从权威性高到低):
| 来源 | 适用 | URL |
|---|---|---|
| 中国信通院 | TMT / AI / 数字化 | caict.ac.cn |
| 国务院发展研究中心 / 国家统计局 | 宏观行业数据 | gov.cn / stats.gov.cn |
| 行业归口部委公开报告 | 行业专项政策 + 规模 | 各部委官网 |
| 艾瑞 / 易观 / 36Kr 研究 | 消费 / TMT / 新业态 | iresearch.com.cn / 36kr.com |
| 赛迪 | 工业 / 制造 / 信息化 | ccidnet.com |
| 头部券商行研(中信 / 中金 / 国君 / 招商) | 资本市场视角 | 各券商研究所 |
| Gartner / IDC / Forrester | 国际视角(找公开摘要) | gartner.com / idc.com |
| 世界级咨询(BCG / McKinsey / Bain) | 战略视角公开报告 | 各官网 insights 页 |

**检索动作**:
- WebSearch:"X 行业 报告 2025"、"X 行业 白皮书 2026"、"X industry report 2025"
- 优先找年份接近的(过去 18 个月内)
- 至少找到 1 份带具体数字的报告

**跳坑提示**:
- 警惕"X 行业研究院"这类不知名机构的报告,可能是商业软文
- 警惕"AI + X"标题党报告,可能没有真实行业数据
- 找不到中文优质报告时,英文报告(McKinsey / BCG)优先于中文劣质报告

### 步骤 2 · 找头部 3 家公司的最新年报 / 招股书 / 公开访谈(20-30 分钟)

**目标**:理解行业在"做生意层面"的真实细节,不是宏观数据。

**为什么 3 家**:1 家是个例,2 家不够立体,3 家(头部 + 跟随 + 颠覆者)能看出行业内部分化。

**找谁**:
- 头部:行业市占率第一(从步骤 1 报告里找)
- 跟随:行业市占率第二/第三
- 颠覆者(如有):新进入者,可能用 AI / 新模式重构行业(找近 3 年新融资公司)

**看什么**:
- 业务结构(主营业务收入占比)
- 客户结构(B / G / C 占比 + 客单价)
- 成本结构(人力 / 原料 / IT 各占比)
- 战略意图(年报致辞 / 招股书"使用募集资金" / 创始人公开访谈)
- AI / 数字化投入(若有)

**信息源**:
- 上市公司:巨潮资讯网 cninfo.com.cn / SEC filings
- 未上市头部:招聘 JD(LinkedIn / 拉勾 / 猎聘)+ 公司官网 + 创始人访谈(36Kr / 虎嗅 / 腾讯科技)
- 行业新闻:近 6 个月该公司被报道的事件

### 步骤 3 · 找行业协会的痛点叙述(15-20 分钟)

**目标**:跳出公司视角,看行业自己怎么定义自己的问题。

**为什么协会很重要**:
- 公司讲故事(讲增长讲战略),协会讲问题(协会要找政策 / 资金 / 标准支持,所以痛点是真痛点)
- 协会的"自我陈述"通常更接近行业从业者的体感

**找谁**:
- 中国 X 行业协会 / X 业商会 / X 工程学会
- 地方性行业协会(深圳 X 协会 / 上海 X 联盟)
- 国际行业协会(若行业全球化)

**看什么**:
- 协会发布的"行业发展白皮书"
- 协会牵头的"标准制定"
- 协会向政府提交的"行业建议书"

**关键问题**:
- 这个行业自己最焦虑的 3 个问题是什么?
- 这些焦虑在公司年报里有没有被提到?(对照公司视角和协会视角的差异)

### 步骤 4 · 找从业者论坛 / 微信公众号 Top 5(20-30 分钟)

**目标**:跳出公司和协会的官方叙事,看一线从业者每天在说什么。

**为什么从业者视角不可替代**:
- 公司讲战略,协会讲政策,**从业者讲痛**——一线痛点最具体、最可操作
- AI 应用要解决的就是一线从业者的痛点

**找谁**:
- 行业垂直微信公众号(用"行业 X + 公众号"搜)
- 知乎该行业话题(知乎搜索 → 按"赞同最高"或"最新"排序)
- 行业垂直论坛(若有)
- B 站 / 抖音 / 小红书的行业从业者账号(尤其新业态)
- 豆瓣小组(部分长尾行业)

**看什么**:
- 一线从业者每天抱怨什么?(琐碎重复劳动 / 信息不通 / 客户难搞 / 流程慢)
- 已经在用什么工具?(传统 IT / Excel / 微信 / 小程序 / SaaS)
- 对 AI 的态度?(期待 / 抵触 / 已经在试)

**纪律**:
- 至少看 5 个不同来源,避免单源偏见
- 收集 5-10 条"具体痛点"原话(不是抽象描述),会成为 AI 应用框架里的灵感

### 步骤 5 · 找 1-2 个失败案例(15-20 分钟)

**目标**:成功案例可以包装,失败案例提供"哪些坑不能踩"的真实警示。

**为什么必须找失败**:
- 该行业的"AI 转型已死"的故事,通常比"AI 转型成功"的故事更接近真相
- 失败案例的根因(技术 / 组织 / 业务 / 战略),可以反向校准本次方案的边界

**找谁**:
- 行业垂直媒体的"复盘 / 反思"类文章
- 36Kr / 虎嗅 / 晚点 / Tech 星球 的"X 公司倒下了 / 转型失败"类深度报道
- 创始人 / 高管在播客 / 公开演讲里的"我们试过 X 但失败了"叙述

**关键问题**:
- 失败的根因是什么?(技术不成熟 / 组织没准备好 / 业务模式不对 / 战略选错时机)
- 早期信号:什么时候本可以止损?
- 对本次咨询的警示

**找不到怎么办**:
- 行业太新没失败案例 → 用跨行业类似场景的失败案例代替("XX 行业当年试过 AI 客服失败,本质问题在 ...")
- 这是常见情况,不要硬编

## 6 · 用户校验机制 · AI 摸底 ≠ 行业认知

**关键纪律**:本框架的产出 A / B 是 AI 在 1-2 小时内能搜到的"公开信息汇总",**不等于行业认知**。摸底完成后必须做用户校验。

**校验三问**(用户必答):
1. **痛点对不对**:产出 A §五 列的 5 个痛点,是不是真痛点?排序合理吗?有没有更痛的没列?
2. **价值链准不准**:产出 A §二 价值链结构,是否反映你/你客户实际的业态?
3. **首发场景靠不靠谱**:产出 B §五 的候选场景,从业者会不会觉得"这是真问题但难解" / "这不是问题"?

**校验时间**:30 分钟即可(用户对照自身行业体感快速过)。

**校验后调整**:
- 用户标注的"不准 / 缺失 / 错位"项 → 回到对应步骤补查
- 用户标注的"完全对" → 进入 [Step 1 探索方向](01-step1-exploration.md) 或直接接 [Step 2 写战略报告](02-step2-drafting.md)

**为什么必须校验**:见 [07-review-thinking-model §3 L2 前提层](07-review-thinking-model.md) —— AI 在公开信息汇总后会自洽,但缺失的关键前提(行业内部真实痛点 / 客户决策真实流程)只能靠用户验证。

## 7 · 衔接到 Step 1 / Step 2

### 7.1 衔接到 Step 1 · 探索方向

```
陌生行业摸底(本框架)
└─ 产出 A 行业知识背景 + 产出 B AI 应用框架
   ↓ 用户校验三问
   ↓ 通过
   ↓
Step 1 · 探索方向 ([01-step1-exploration.md](01-step1-exploration.md))
├─ Step 1.1 行业速览  → 已由本框架产出 A 完成,可直接复用
├─ Step 1.2 全量候选 12-20 个  → 在产出 B §一 的 8-15 个基础上扩展到 12-20 个,补国际对标
└─ Step 1.3 Top 3 推荐  → 给客户做评估打分,等用户选定
```

**总时长**:陌生行业摸底 1-1.5 小时 + 校验 30 分钟 + Step 1 扩展 + Top 3 评估 = **2-3 小时**到 Step 1 产出

### 7.2 衔接到 Step 2 · 写战略报告

```
陌生行业摸底(本框架)
└─ 产出 A 行业知识背景 + 产出 B AI 应用框架
   ↓ 完成 Step 1 探索 + 用户选定 Top 推荐
   ↓
Step 2 · 写战略报告 ([02-step2-drafting.md](02-step2-drafting.md))
├─ 第 1 章执行摘要 / 第 2 章背景诊断  → 主要用产出 A
├─ 第 3 章 AI 战略主线 + 第 4 章 Top 推荐方向  → 主要用产出 B + Step 1 评估结果
├─ 第 5 章不做什么 + 第 6 章路线图  → 写阶段产生
└─ 第 7 章风险与边界  → 在产出 A §五 风险条目基础上扩展
```

**总时长**:陌生行业摸底 1-1.5 小时 + Step 1 2-3 小时 + Step 2 8-12 小时(含 Checkpoint 暂停) = **完整战略报告 2-3 天**

### 7.3 是否启用可选 add-on

陌生行业摸底完成后,在 Step 1 客户上下文采集环节确认:
- 客户是否需要 ROI 测算? → 启用 [add-on-roi.md](add-on-roi.md)
- 客户是否需要政策合规章节? → 启用 [add-on-soe-policy.md](add-on-soe-policy.md)

**默认对世界前沿科技公司类客户**:两个 add-on 都不启用,直接对标 Google / Anthropic / OpenAI 等公司的 AI 战略叙事方式。

## 8 · 框架的边界

本框架适合:
- 用户提出的行业不在 6 套样板库中
- 用户对该行业**不算专业**(只是听说 / 客户在做 / 想探索)
- 1-2 小时内可以出"够用的认知"

**本框架不适合**:
- 用户已经是该行业资深从业者(应直接走 Step 1 / Step 2,把自身行业体感作为客户上下文采集的私域信息)
- 涉及国家机密 / 高度合规敏感行业(如军工 / 核能 / 部分金融子赛道) —— 公开信息覆盖度有限,需要客户内部专家深度参与
- 极度小众且无任何公开数据的行业(如某些非遗手工业) —— 摸底框架可能产出 < 1 页,需要主要靠用户输入

**遇到不适合场景的处理**:在产出 A 顶部明示"本摸底基于公开信息,深度受限,请用户/行业专家补充验证",不假装摸到位。

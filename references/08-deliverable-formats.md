## 2 · 主报告（11 章 60-100 页）

### 2.1 字数控制

| 章 | 字数 | 页数（11pt 仿宋）|
|---|---|---|
| 0 | 500-800 | ≤ 1 |
| 1 | 1500-2500 | 3-5 |
| 2-9 | 各 2500-8000 | 各 5-15 |
| 10 | 1500-2500 | 3-5 |
| **总计** | **31000-53000** | **62-108** |

详细见 [03-report-architecture.md](03-report-architecture.md)。

### 2.2 模板见 assets

[assets/main-report-template.md](../assets/main-report-template.md) 提供 11 章完整骨架 + 占位段落 + 图表位置标记。

### 2.3 输出格式

- **Markdown 源稿**：`XX-战略报告-主文.md`
- **Word 版本**：`XX-战略报告-主文.docx`（蓝橙配色 + 公文字体）
- **PDF 版本**：从 Word 导出（保留高清图表）

## 3 · 一页纸（董事长入口）

### 3.1 字数与时长

- 字数：200-400 字
- 讲解时长：30 秒
- 阅读时长：1-2 分钟

### 3.2 结构 SCR + 4 象限

```
┌──────────────────────────┬──────────────────────────┐
│ 战略建议 + 时间窗         │ 三项约束（外部客观）       │
│                          │                          │
│ [一句话建议]              │ ◯ 政策窗口（具体文件）     │
│ [起步时间]                │ ◯ 竞争格局（X 层空间）    │
│ [模式定调]                │ ◯ 自身禀赋（资质/数据）    │
│                          │                          │
│                          │ —— 均为外部客观条件        │
├──────────────────────────┼──────────────────────────┤
│ 投入回报（基准档）         │ 第一步                   │
│                          │                          │
│ 三年投入：X 万            │ MVP 周期：6-10 周          │
│ L1+L2 三年回报：Y 万       │ 外部支出：N 万             │
│ 现金 ROI：N1%             │ 内部抽调：M 名工程师        │
│ 综合 ROI：N2%             │ MVP 通过后再启动产品化      │
│ 回本：N3 / N4 个月        │                          │
└──────────────────────────┴──────────────────────────┘
```

## 4 · PPT 汇报方案（20 页正文 + 6 附录）

### 4.1 时长

- 总汇报：25-30 分钟讲解 + 15-20 分钟 Q&A
- 短场景：10-15 分钟董事会插汇 → 压缩至 12 页（见 ppt-outline-template）

### 4.2 页面结构

| 页 | 主题 | 时长 |
|---|---|---|
| 1 | 封面 | 30s |
| **2** | **一页纸结论**（4 象限） | **2-3 分** |
| 3 | 为什么是现在（三圆相交）| 1.5 分 |
| 4 | 战略空间（1+N+X 架构）| 1 分 |
| 5 | 战略主线全景（四层架构总图）| 2 分 |
| 6-7 | 双产品线首发场景 | 3.5 分 |
| 8 | 共享底座 | 1.5 分 |
| 9 | MVP 6-10 周路径 | 2 分 |
| **10** | **投入回报**（三档 × 四层矩阵）| **2-3 分** |
| 11 | 客户与商业模式 | 1.5 分 |
| 12 | 组织与人员（曲线对比）| 1.5 分 |
| 13 | 政策对接与大赛 | 1 分 |
| 14 | 风险与边界（红线表）| 1.5 分 |
| 15 | 三年里程碑（甘特图）| 1 分 |
| 16 | L4 软实力（雷达图）| 1 分 |
| **17** | **启动前三项决策**（清单卡）| **2 分** |
| 18 | 一句话总结 | 1 分 |
| 19 | Q&A 引导 | — |
| 20 | 致谢 | — |
| A1-A6 | 附录 · 详细 ROI / 竞争对比 / 备选方案 / 客户画像 / 硬件 / MVP 指标 | Q&A 翻看 |

### 4.3 关键页画面规范

详见 [assets/ppt-outline-template.md](../assets/ppt-outline-template.md)，含第 2/5/10/16/17 五张关键页的详细画面构想 + 讲解话术。

### 4.4 输出格式

- **Markdown 大纲**：`XX-PPT方案.md`
- **PPT 文件**：`XX-PPT方案.pptx`（用 document-skills:pptx 生成）
- **关键页 PNG**：用 diagram-gen 生成（蓝橙配色，3x DPI）

## 5 · 6 类附件

### 5.1 附件 A1 · 详细 ROI 测算

**字数**：2000-3000

**结构**：
- 三档投入展开表（按类别）
- 三档回报展开表（按 L1-L4）
- 客户单价分档说明
- 三年签约漏斗
- 敏感性分析（关键变量 ±20%-50%）
- 一句话口径

**模板见**：[05-roi-templates.md](05-roi-templates.md)

### 5.2 附件 A2 · 竞争格局对比

**字数**：2000-3000

**结构**：
- 至少 5 家对手 × 6 维对比矩阵
- 维度：产品形态 / 技术栈 / 价格区间 / 主要客户 / 政策资质 / 公开案例
- 每家对手 1 段定性分析

### 5.3 附件 A3 · 备选方案与不选原因

**字数**：1000-2000

**结构**：
- 至少 2 个备选战略（如方案 B 重资产 / 方案 C 联合开发）
- 每个备选的优劣对比
- 每个不选的具体理由（量化为主）

### 5.4 附件 A4 · 客户画像

**字数**：1500-2500

**结构**：
- 至少 3 类目标客户画像（决策者 / 使用者 / 影响者）
- 每类客户的痛点 / 决策链 / 触达方式

### 5.5 附件 A5 · 硬件 / 合作进度（如适用）

**字数**：500-1500

**结构**：
- 软硬件合作伙伴的洽谈状态
- 关键合作的法律 / 商务条款
- 已签约 / 在谈 / 待谈 三类

**注**：如客户战略不涉及硬件 / 合作，本附件可省略。

### 5.6 附件 A6 · MVP 评估指标体系

**字数**：1000-2000

**结构**：
- MVP 通过的所有量化指标（查准率 / 召回率 / 时延 / 工时节约）
- 评测方法（数据集 / 评测人 / 评测周期）
- 数据集来源与脱敏方式

## 7 · 风格与配色规范

### 7.1 蓝橙双色系（推荐 · 公文风格）

| 用途 | 颜色 | RGB |
|---|---|---|
| 主色（标题 / 一级强调） | #1D4ED8 蓝 | (29, 78, 216) |
| 辅色（数据 / 二级强调） | #C2410C 橙 | (194, 65, 12) |
| 主文字 | #475569 深灰 | (71, 85, 105) |
| 弱文字 | #64748B 灰 | (100, 116, 139) |
| 极弱文字 | #94A3B8 浅灰 | (148, 163, 184) |
| 强调白底 | #FFFFFF 白 | (255, 255, 255) |

### 7.2 字体（公文风格）

| 用途 | 字体 | 字号 |
|---|---|---|
| 一级标题 | 方正小标宋简体 | 22 |
| 二级标题 | 黑体 | 16 |
| 三级标题 | 楷体 | 14 |
| 四级标题 | 仿宋（粗）| 12 |
| 正文 | 仿宋 | 11-12 |
| 引用 | 楷体（斜）| 11 |
| 表格 | 仿宋 | 10 |

### 7.3 表格规范

- 表头：蓝色填充 + 白字
- 表体：白底 + 黑字
- 表格四边框 + 行间细线
- 重要单元格加粗

### 7.4 图表规范

- 主色用 #1D4ED8 蓝
- 辅色用 #C2410C 橙
- 第三色用 #64748B 灰
- 不用花哨的渐变 / 阴影 / 立体效果
- 字体与正文统一（仿宋 / 黑体）

### 7.5 替代风格

如客户偏好其他配色（红色系 / 绿色系），用 theme-factory skill 切换主题。

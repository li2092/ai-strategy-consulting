## 0 · 前置 · 客户上下文 7 项 checklist（必须先做）

启动五维采集前，先在 30 分钟内确认 7 件事。任何一项没确认就开工，后续返工率 ≥ 50%。

### 0.1 客户画像（4 项）

| 项 | 关键问题 | 影响 |
|---|---|---|
| **身份** | 客户是谁？所属行业、规模、价值链上的位置？ | 决定术语颗粒度与论证深度 |
| **决策人** | 最终拍板的人是谁？职级、技术背景、最关心什么？ | 决定语体与重点 |
| **痛点起点** | 是什么事件触发了这次咨询？ | 报告必须命中此痛点，不是相邻问题 |
| **政治格局** | 谁会受益、谁会抗拒、权力关系如何？ | 决定建议表达方式 |

### 0.2 交付物规格（3 项）

| 项 | 关键问题 |
|---|---|
| **类型** | 战略报告 / 路线图 / 评估 / 提案？|
| **长度 + 形态** | 主报告 60-100 页 / 一页纸 / PPT 三件套？还是只要一项？|
| **语言** | 中文 / 英文 / 双语？目标读者是国内国企还是跨国总部？|

### 0.3 写作者身份（必须最先明确）

> **谁是稿件中的"我们"？**

| 身份 | 语气示例 | 典型场景 |
|---|---|---|
| 外部咨询顾问 | "基于我们对客户的评估..." | 客户付费咨询稿件 |
| 内部改革小组 | "我们识别出..." | 公司战略文档 |
| 技术供应商 | "我们的方案能解决..." | 产品提案 |
| 中立分析师 | "分析显示..." | 独立评估 |

**纪律**：身份未确认前不得动笔。用错"我们"会让整篇报告语气崩盘。

## 1 · 五维输入总表

| # | 维度 | 关键问题 | 必备产出 | AI 生成内工时占比 |
|---|---|---|---|---|
| 1 | 客户自身 | 客户的钱、人、技术、决策怎么分布？ | 客户画像 1 份 | 25% |
| 2 | 政策环境 | 客户业务受哪些政策约束？哪些是窗口期？ | 政策图谱 1 份 | 15% |
| 3 | 行业现状 | 客户在行业里的位置？行业向哪走？ | 行业地图 1 份 | 20% |
| 4 | 学术前沿 | 我们想做的 AI 应用在学术上跑通了吗？ | 5 主题地图 1 份 | 20% |
| 5 | 跨行业 AI 实践 | 别的行业类似场景怎么落地的？ | 案例集 1 份（5 成功 + 3 失败）| 20% |

**时间预算**:
- **AI 生成**:1-2 小时(由 Claude / 类 AI 工具完成五维素材汇总 + 起草 5 份产出物)
- **用户审稿**:0.5 天(每维 30-60 分钟,确认事实 / 补充行业理解 / 标注【需问客户】项)

**纪律**:不要因为"AI 生成快"就跳过审稿。审稿是质量底线,不是可压缩项。
- 五维任何一维"未审"就推进 Mode P,后续 Planning 会建立在错误前提上
- 用户对行业的体感和私域信息是 AI 替代不了的(见 §0.1 客户上下文 7 项)
- 陌生行业(不在 §8 样板库)需追加 1-2 小时跑陌生行业摸底框架(见 [12-unfamiliar-industry-framework.md](12-unfamiliar-industry-framework.md))

## 2 · 维度一 · 客户自身

### 2.1 必采集要素

| 要素 | 优先级 | 公开来源 | 私域信息（需问用户） |
|---|---|---|---|
| 公司基本盘（成立年、注册资本、控股股东、注册地） | P0 | 工商查询、企查查、官网 | — |
| 业务结构（主营业务、收入占比、客户类型） | P0 | 年报、招股书 | 业务真实占比、未披露收入 |
| 技术现状（IT 系统、研发人员、技术栈） | P1 | 招聘 JD、新闻、专利 | 实际系统数、技术债 |
| 战略意图（董事长讲话、战略报告、新闻发布） | P0 | 党建报道、年报致辞、新闻 | 决策时间窗 |
| 决策链（董事长、总经理、CTO/CIO、其他决策人） | P1 | 官网团队、政府公示 | 实际权力关系、内部派系 |
| 资质资产（牌照、资质证书、专利、知识产权） | P0 | 官网、政府公示 | 资质实际激活状态 |

### 2.2 行业差异化采集策略

| 行业类型 | 采集重点 | 关键差异 |
|---|---|---|
| **国企/央企** | 党建报道 + 年报 + 党建系统党委决策 + 国资委文件 | 决策慢、合规重、看政治指标 + 经济指标 |
| **民营上市** | 招股书 + 投资者关系 + 财报电话会 | 看市值管理 + 业务增长 |
| **创业公司** | 融资轮次 + 招聘 JD + 官网产品 | 看融资节奏 + 产品迭代 |
| **政府事业单位** | 政府工作报告 + 部门预算 + 招标信息 | 看预算盘子 + 政策对接 |

### 2.3 检索工具

| 工具 | 用途 | URL |
|---|---|---|
| 工商企查 | 公司基本盘 + 关联交易 + 司法风险 | qcc.com / qichacha.com |
| WebSearch | 新闻、公告 | 工具内置 |
| 招聘网站 | 技术栈 + 团队规模 | linkedin.com / liepin.com / lagou.com |
| 政府公示 | 资质、牌照 | 行业部委官网 |

### 2.4 客户画像产出模板

```markdown
# 01-客户画像

## 一、基本盘
- 公司全称 / 简称 / 成立年 / 注册资本
- 控股股东 / 实控人 / 股权结构
- 总部地址 / 主要办公地 / 分支机构

## 二、业务结构
- 主营业务（按收入排序）
- 业务模式（B2B / B2G / B2C）
- 客户类型 + Top 5 客户占比【推断】
- 收入规模 / 利润率【事实 + 推断】

## 三、技术现状
- IT 系统现状（已部署 / 计划中）
- 研发人员规模【推断】
- 主要技术栈
- AI / 数据科学相关人员【需问】

## 四、战略意图
- 公开战略口径（来自年报致辞 / 党委决议 / 战略发布会）
- 当前重点战役【推断】
- 决策时间窗【需问】

## 五、决策链
- 董事长 / 总经理 / 主管副总
- 关键决策人画像
- 内部派系与权力关系【需问】

## 六、资质资产
- 牌照 / 资质证书 / 专利
- 数据资产
- 已部署的 IT 平台

## 七、未确认信息清单（必填）
- [ ] 业务真实结构（公开数据 vs 实际占比）
- [ ] 内部决策时间窗
- [ ] 关键决策人对 AI 的态度
- [ ] ...
```

## 3 · 维度二 · 政策环境

### 3.1 必采集层级

| 层级 | 内容 | 来源 |
|---|---|---|
| **国家层级** | 国务院文件、五年规划、年度政府工作报告 | gov.cn |
| **部委层级** | 工信部、发改委、网信办、行业部委 | 各部委官网 |
| **行业层级** | 行业协会标准、行业白皮书 | 行业协会官网 |
| **地方层级** | 省/市政府文件、园区政策 | 地方政府官网 |
| **AI 专项** | 大模型登记、生成式 AI 服务管理办法 | cac.gov.cn |

### 3.2 关键检索时间窗

- **当前+过去 18 个月**的所有相关政策（更早的容易过时）
- **当前+未来 36 个月**的考核节点（如"2027 年智能化普及率 70%"这类时点）

### 3.3 政策影响分析

每条政策必须分析三层影响：
- **强制影响**：必须做的事（合规性）
- **激励影响**：做了会有什么红利（资金 / 项目机会）
- **窗口期**：何时窗口关闭

## 5 · 维度四 · 学术前沿

### 5.1 主题地图（5 大主题 × 各 5-10 篇）

按 AI 战略可能覆盖的方向，分 5 大主题做学术综述：

| 典型主题（视客户行业调整） | 检索关键词示例 |
|---|---|
| RAG / 知识库 | "retrieval augmented generation" + "domain" + "2026" |
| Agent 编排 | "multi-agent" + "tool use" + "planning" |
| 行业大模型 / LoRA 微调 | "domain-specific LLM" + "LoRA" + "industry" |
| 视觉识别（适合工程基建/制造）| "object detection" + "defect" + "industrial" |
| 知识图谱 + LLM | "knowledge graph" + "LLM" + "reasoning" |

### 5.2 学术资源

| 资源 | 用途 | URL |
|---|---|---|
| arXiv | 最新论文（含未发表）| arxiv.org |
| Semantic Scholar | 论文检索 + 引用图谱 | semanticscholar.org |
| Papers with Code | SOTA 跟踪 | paperswithcode.com |
| OpenAlex | 学术图谱 | openalex.org |
| Google Scholar | 综合检索 | scholar.google.com |
| 顶会列表 | NeurIPS / ICML / ACL / CVPR / EMNLP | 各会议官网 |

### 5.3 工业级落地信号

学术 SOTA 不等于可商业化。判断"可商业化阈值"：
- 有 1+ 公司部署案例（含失败案例）
- 至少有 1 个开源工业级实现
- 至少 1 个行业的客户已签约付费

如果三项都缺，标记为【实验阶段，2 年内不建议商业化】。

### 5.4 学术地图产出模板

```markdown

- 论文 2 ...
- 论文 5-10 ...

- 公司 A 部署：...

- 直接可用 / 需要改造 / 不建议商业化
```

## 6 · 维度五 · 跨行业 AI 实践

### 6.1 案例采集要求

至少 5 个**跨行业**成功案例 + 至少 3 个**失败案例**。

为什么必须有失败案例：成功案例可以包装，失败案例提供"哪些坑不能踩"的警示。

### 6.2 案例来源

| 来源 | 类型 | URL |
|---|---|---|
| BCG / McKinsey / Bain 公开报告 | 战略级案例 | bcg.com / mckinsey.com / bain.com |
| MIT Sloan Management Review | 工业级案例 + 学术分析 | sloanreview.mit.edu |
| HBR | 转型案例 | hbr.org |
| Palantir / Snowflake / Databricks 客户案例 | 平台级案例 | 各公司官网 customer stories |
| OpenAI / Anthropic 客户案例 | LLM 应用案例 | openai.com/customer-stories / anthropic.com/customers |
| Gartner Magic Quadrant | 行业全景 | gartner.com（找公开摘要）|

### 6.3 案例分析模板

```markdown
# 案例 N · [公司名 + 行业]

## 背景
- 客户行业 / 规模 / AI 投入
- 触发 AI 应用的痛点

## 方案
- 应用方向（RAG / Agent / 视觉 / 等）
- 技术栈
- 实施周期
- 投入规模

## 结果（必带数据）
- ROI / 工时节约 / 收入增长
- 时间到见效

## 启示
- 对本客户的可借鉴点
- 不可借鉴点（行业差异 / 规模差异 / 政策差异）
```

### 6.4 失败案例必备分析

每个失败案例必须分析：
- 失败的根因（技术 / 组织 / 业务 / 战略）
- 早期信号（什么时候应该止损）
- 对本客户的警示

## 7 · 信息缺口检测与提问模板

### 7.1 信息缺口分类

| 类型 | 表现 | 处理方式 |
|---|---|---|
| **公开资料能查但暂未查** | 漏检索；时间不够 | 继续查，不问用户 |
| **公开资料无法回答** | 客户私域信息 / 内部数据 | **必问用户** |
| **回答存在多种可能** | 战略意图 / 决策时间窗 | **必问用户** |
| **超出当前 Mode 范围** | Mode D 阶段问 ROI 细节 | 标记到下一 Mode 处理 |

### 7.2 提问纪律

- **一次问 ≤ 3 个问题**（用户认知负荷有限）
- **优先级排序**：能直接卡住下一步工作的问最先
- **场景化**：解释为什么需要这个信息（让用户明白回答的代价）
- **给候选答案**：可能的话给 2-3 个选项让用户挑（比开放问题更易回答）
- **深夜确认**：用户在 22:00 后回答，确认是否次日仍有效

### 7.3 提问模板

**模板 A · 客户私域信息**
```
为推进 [当前任务]，需向您确认几项 [客户名] 内部信息：
1. [问题 1] —— 用于 [具体用途]
2. [问题 2] —— 用于 [具体用途]
3. [问题 3] —— 用于 [具体用途]

如果 1-2 周内无法获得，建议改为 [备选方案]（用 [其他角度的公开数据] 替代）。
```

**模板 B · 战略意图确认**
```
本报告的关键设计取决于 3 个意图：
1. 主要读者是？(a) 董事长 (b) 全体高管 (c) 政府 / 投资人 (d) 内部团队
2. 决策时间窗是？(a) 本月 (b) 本季度 (c) 半年内 (d) 一年内
3. 报告用途是？(a) 战略决策 (b) 政府备案 (c) 招投标 (d) 内部沟通

如果 (a)+(a)+(a)，报告将按 [模板 X] 起草；如果 (b)+(c)+(b)，按 [模板 Y]。请确认。
```

**模板 C · 行业专有约束**
```
[客户行业] 的合规约束超出了我目前的资料覆盖：
1. [子赛道 A] 是否需要 [资质 B]？
2. [业务 C] 是否受 [监管 D] 约束？

如果您不清楚，可指定 [客户内部相关方]（法务 / 合规部）确认。
```

## 8 · 采集 checklist 样板（10 套 · 按"客户类型 4 套 + 行业类型 6 套"组织）

> **样板,不是边界**:本节列出 10 套常用样板,客户行业不在其中(如宠物 / 殡葬 / 印刷 / 民营养老 / 二手奢侈品 / 任意长尾业态)→ 必须先跑 [12-unfamiliar-industry-framework.md](12-unfamiliar-industry-framework.md) 5 步法,产出该行业的"知识背景 + AI 应用框架"再进入 Mode D / Mode L。

### 一组 · 按客户类型(4 套)

#### 8.1 国企/央企客户 · checklist

### 二组 · 按行业类型(6 套样板)

> 每套含三块元数据:**信息源 · 典型痛点 · ROI 锚点**。锚点为量级参考,实际须按客户付费能力回归。

#### 8.5 医疗(医院 / 诊所 / 医美 / 药企)· checklist

- **信息源**:国家卫健委(nhc.gov.cn) / 国家药监局(nmpa.gov.cn) / 中国医院协会 / 中华医学会 / 健康界 / 八点健闻
- **典型痛点**:① 医生经验留不住(高级医师退休 → 经验断层) ② 病历质控人力不够 ③ 患者随访 / 复诊率低 ④ 集采政策下营收压力大 ⑤ DRG/DIP 付费改革下成本控制要求高
- **ROI 锚点**:三甲医院单项目 100-500 万 / 二级医院 30-100 万 / 民营诊所 5-30 万 / 医美连锁 20-80 万 · 综合 ROI 200-400%

- [ ] 医院等级 + 床位数 + 年门诊量
- [ ] 信息化系统现状(HIS 医院信息系统 / EMR 电子病历 / PACS 影像 / LIS 检验)
- [ ] 卫健委评级与考核指标
- [ ] 药品 / 器械集采政策影响
- [ ] 互联网医院牌照(若有)
- [ ] DRG/DIP 付费改革对该医院影响
- [ ] 医生人数 + 主任医师占比 + 流动率【需问】

#### 8.6 教育(高校 / K12 / 培训机构)· checklist

- **信息源**:教育部(moe.gov.cn) / 各省教育厅 / 中国教育学会 / 多鲸资本 / 芥末堆 / 蓝鲸教育
- **典型痛点**:① 双减 / 反学科类培训政策下生存压力 ② 教师培养周期长 ③ 个性化教学难大规模实施 ④ 高校就业率压力 ⑤ 招生 / 续报转化率低
- **ROI 锚点**:重点高校单项目 50-200 万 / K12 集团 20-100 万 / 单培训机构 5-30 万 / 在线教育上市公司 100-500 万 · 综合 ROI 200-400%

- [ ] 办学类型(公办 / 民办 / 培训机构)+ 学生规模
- [ ] 教育部 / 教育厅相关政策(尤其双减及最新文件)
- [ ] 已部署的教学/管理系统(教务 / 学习管理 / 在线教学)
- [ ] 师资规模 + 学生续报率 + 招生转化率【需问】
- [ ] 与教育主管部门项目合作历史
- [ ] 是否上市 / 是否为关联港美股教育公司

#### 8.7 制造(离散制造 / 流程制造 / 装备 / 电子)· checklist

- **信息源**:工信部(miit.gov.cn) / 国家市场监管总局 / 中国机械工业联合会 / 各细分协会(如汽车工业协会) / 赛迪顾问 / 第一财经
- **典型痛点**:① 老师傅经验流失(质检 / 工艺) ② 设备故障预测难 ③ 排产 / 调度依赖人工经验 ④ 跨工厂数据孤岛 ⑤ 产品质量缺陷追溯链长
- **ROI 锚点**:大型制造集团单项目 200-1000 万 / 中型上市公司 50-200 万 / 中小制造企业 10-50 万 · 综合 ROI 250-500%(制造业 AI 提效空间大)

- [ ] 制造类型(离散 / 流程)+ 产线数 + 年产值
- [ ] 已部署系统(ERP / MES 制造执行 / WMS 仓储 / SCADA 监控)
- [ ] 智能制造试点项目(国家级 / 省级)
- [ ] 设备数字化率(联网设备占比)【需问】
- [ ] 工程师团队规模 + IT 部门人数
- [ ] 上下游供应商数字化协同程度【需问】

#### 8.8 物流(快递 / 货运 / 仓储 / 园区)· checklist

- **信息源**:国家邮政局(spb.gov.cn) / 交通运输部 / 中国物流与采购联合会(CFLP) / 罗戈网 / 物流时代周刊 / 运联智库
- **典型痛点**:① 末端配送成本高 ② 仓储周转效率低 ③ 路径规划依赖人工经验 ④ 包裹分拣错误率 ⑤ 客户咨询响应慢
- **ROI 锚点**:头部快递 / 物流集团单项目 100-500 万 / 区域物流龙头 30-100 万 / 中小物流公司 10-30 万 / 物流园区 50-200 万 · 综合 ROI 250-450%

- [ ] 业务类型(快递 / 干线货运 / 同城 / 仓储 / 园区运营)
- [ ] 日均订单量 / 仓库面积 / 车辆数
- [ ] 已部署系统(WMS 仓储 / TMS 运输 / OMS 订单 / 调度系统)
- [ ] 自有运力 vs 外包运力比例【需问】
- [ ] 行业政策(数据合规 / 危化品 / 跨境)
- [ ] 与电商 / 制造客户的接入深度【需问】

#### 8.9 零售(线下连锁 / 电商 / 品牌 / 商超)· checklist

- **信息源**:商务部(mofcom.gov.cn) / 中国连锁经营协会(CCFA) / 中国零售业协会 / 36Kr / 联商网 / 亿邦动力
- **典型痛点**:① 选品 / 补货依赖经验 ② 客户复购率低 ③ 门店运营标准化难 ④ 直播 / 内容运营人力不够 ⑤ 跨渠道库存与价格不统一
- **ROI 锚点**:连锁百强 100-500 万 / 区域连锁 30-100 万 / 单品牌 / 单店 5-30 万 / 头部电商 200-1000 万 · 综合 ROI 200-400%

- [ ] 业态(线下 / 电商 / 全渠道)+ 门店 / SKU 数
- [ ] POS / ERP / CRM / WMS 系统现状
- [ ] 主流电商平台店铺数(天猫 / 京东 / 抖音 / 小红书 / 拼多多)
- [ ] 自营私域(小程序 / 企微 / 社群)规模
- [ ] 平均客单价 + 复购率【需问】
- [ ] 直播 / 内容团队规模【需问】

#### 8.10 农业(种植 / 养殖 / 农资 / 农产品流通)· checklist

- **信息源**:农业农村部(moa.gov.cn) / 国家统计局农业数据 / 中国农业产业化龙头企业协会 / 农民日报 / 35 斗(垂直农业科技媒体)
- **典型痛点**:① 病虫害识别依赖经验 ② 农产品分级与定价标准化难 ③ 上下游信息不通(种植户 ↔ 加工 ↔ 流通) ④ 高级农技人才稀缺 ⑤ 农产品销售渠道分散
- **ROI 锚点**:大型农企 50-200 万 / 合作社 10-50 万 / 个体农户(几乎不付费,需要政府/平台补贴) 1-10 万 / 农资上市公司 100-300 万 · 综合 ROI 200-350%(农业付费意愿低,需结合政府补贴)

- [ ] 业态(种植 / 养殖 / 农资 / 流通)+ 规模(亩 / 头 / SKU)
- [ ] 已部署系统(ERP / 物联网传感器 / 溯源系统)
- [ ] 农业农村部 / 省厅相关补贴政策
- [ ] 数字农业试点项目(国家级 / 省级)
- [ ] 与电商 / 商超 / 加工厂的接入合作【需问】
- [ ] 实际付费方判断(农户 / 合作社 / 政府 / 平台 / 加工厂)【需问】

### 8.11 不在 10 套样板内的行业怎么办

→ 必须先跑 [12-unfamiliar-industry-framework.md](12-unfamiliar-industry-framework.md) 的 5 步法,产出"行业知识背景 + AI 应用框架",再进入 Mode D / Mode L。

不要硬把陌生行业塞进相近样板("宠物医院 ≈ 医疗" / "民营养老 ≈ 医疗" / "二手奢侈品 ≈ 零售" / "电竞陪练 ≈ 教育")—— 不同业态的价值链 / 客户决策链 / 政策约束差异巨大,套错样板比没样板更糟。
